【归纳】函数的防抖与节流

广宇 2019-3-13 JavaScript 面经

再编辑于 2020年1月19日 13:49:40

现在临近春节，手上的活稍微轻松了一些，想着把之前写得比较水的博客整理趁这个时候整理一下。结果发现之前学习的 Vue 的内容基本忘得很干净，真是哀其不幸怒其不争。正好看到这篇博客写得比较简单，当时应该是在面试被问到这个问题吃瘪了，就想着赶紧记下来，那时候也比较焦虑，写博客实在也没有什么心思。其实在当时就对防抖节流这部分不大懂，这回正好把这部分内容复习一下了。

前几天正好看了防抖与节流的相关内容，没想到面试就被问到了，当时没有总结这个知识点。现在在这里简单地归纳一下吧。防抖与节流都是属于防止函数过度调用的手段。在平时开发的过程中，又是会需要频繁调用一些函数，比如用户滚动到某处页面，计算滚动的距离之类的。如果直接监听用户的滚动事件，并输出滚动距离的话，那函数的调用时非常频繁的。这样不仅会加重浏览器的负担，也会影响用户体验。因此对防抖和节流的了解还是必要的

# 一、函数防抖（debounce）

所谓防抖，就是当持续触发事件时，一定时间段内没有再触发事件，事件处理函数才会执行一次，如果设定的时间到来之前，又一次触发了事件，就重新开始延时。打个比方，就像是在抢票软件上抢票，假设我固定需要 10 天才能抢到票，但是我在抢票期间想改票，那就得重新抢票，重新计时。下面就是一个简单的例子：

var timer = null;
function debounce(){
    clearTimeout(timer); // 清除未执行的代码，重置回初始化状态
    timer = setTimeout(function(){
        console.log("函数防抖");
    }, 300);
};

在这个例子中，触发debounce之后首先会清除定时器，再设置一个定时器 300 ms 后触发函数。在这 300 ms 内如果再次调用debounce也不会多次执行 定时器中的函数，起到了防止频繁触发的效果。

当然，函数防抖还有分为两种不同的方式，函数可以选择在定时开始前，或者之后定时结束后触发。刚才的例子显然是在定时结束后触发的。如果设置在定时前执行函数，那么在计时时间内多次触发，同样也不会执行函数，并重新计时，直到定时结束之后触发事件才会执行函数。在平时可以经常用到的 lodash 中，也提供了配置更为灵活的防抖方法debounce，可以使用两种不同的防抖方式。下面是来自 lodash 的介绍：

创建一个 debounced（防抖动）函数，该函数会从上一次被调用后，延迟 wait 毫秒后调用 func 方法。 debounced（防抖动）函数提供一个 cancel 方法取消延迟的函数调用以及 flush 方法立即调用。 可以提供一个 options（选项） 对象决定如何调用 func 方法，options.leading 与|或 options.trailing 决定延迟前后如何触发（是 先调用后等待 还是 先等待后调用）。 func 调用时会传入最后一次提供给 debounced（防抖动）函数 的参数。 后续调用的 debounced（防抖动）函数返回是最后一次 func 调用的结果。

下面是一个例子，func似乎被设置为既会在定时前执行，也会在定时结束后执行，但实际上并非如此。点击「执行函数」后，func执行，并开始 2000 ms 的计时。此时再次点击「执行函数」，func不会执行且计时被重置，但是累计的等待时间不超过 3000 ms，计时结束后触发func一次。这里的「在定时结束后执行」规定的是在计时阶段内debounce的行为，如果当前不在计时，则视为debounce是在定时开始前执行，如果单纯只执行一次debounce，func并不会在定时前后分别执行一次：

function Debounced() {
  const debounce = lodash.debounce(() => console.log('函数防抖'), 2000, { maxWait: 3000, leading: true, trailing: true });
  return (
    <div>     
      <button onClick={this.debounce}>执行函数</button>
      <button onClick={this.debounce.flush}>停止计时</button>
      <button onClick={this.debounce.cancel}>立即调用</button>
    </div>
  )
}

此外，debounce还提供了cancel和flush方法，cancel方法会终止当前正在计时的debounce的行为，销毁定时器；flush会使当前正在计时的定时器立即结束，如果设定func是在定时后调用，func就立即执行了。

# 二、函数节流（throttle）

函数节流则是另一种防止函数频繁触发的手段。与会重新计时的防抖不同，节流规定的是一段时间内函数最多之内被执行一次，即当持续触发事件时，保证一定时间段内只调用一次事件处理函数。这与游戏中的「冷却时间」类似，「冷却时间」没有结束就放不了技能。下面是一个简单的例子：

var cd = false;
function throttle(){
  if(cd){
    // 判断是否已空闲，如果在执行中，则直接return
    return;
  }
  timeId = setTimeout(function(){
    console.log("函数节流");
    cd = false;
  }, 300);
  cd = true;
};

在上面的例子中，只有当前不在「冷却时间」内，事件处理函数才会开始新的计时，计时结束后执行console.log。与防抖类似，节流也分为在计时开始前执行函数和在计时结束后执行这两种，这里只要把console.log移到定时器外面就可以实现在计时开始前执行的效果。在 lodash 里也同样提供了一种throttle方法，与debounce需要接受参数也基本类似：

创建一个节流函数，在 wait 秒内最多执行 func 一次的函数。 该函数提供一个 cancel 方法取消延迟的函数调用以及 flush 方法立即调用。 可以提供一个 options 对象决定如何调用 func 方法， options.leading 与|或 options.trailing 决定 wait 前后如何触发。 func 会传入最后一次传入的参数给这个函数。 随后调用的函数返回是最后一次 func 调用的结果。

下面则是使用lodash.throttle的例子，与lodash.debounce相似，设置leading和trailing均为true会使得func会在计时开始前执行一次，如果在计时过程中还点击了「执行函数」，计时结束后也会执行一次func。但是与debounce不同，这里leading和trailing的默认值都是true，而debounce中只有trailing是默认为true的。如果设置任何一个为false，在计时阶段调用throttle不会有任何效果。

function Throttle() {
  const throttle = lodash.throttle(() => console.log('函数节流'), 2000); // 默认 { leading: true, trailing: true }
  return (
    <div>     
      <button onClick={this.throttle}>执行函数</button>
      <button onClick={this.throttle.flush}>停止计时</button>
      <button onClick={this.throttle.cancel}>立即调用</button>
    </div>
  )
}
