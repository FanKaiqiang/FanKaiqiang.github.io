【归纳】BFC解惑

广宇 2018-10-16 CSS

今天决定再肝一篇博客，打算在学校的这几天把CSS深入浅出的知识做一个完整的总结。这次的内容就是BFC了，学了BFC应该也有一两个星期没复习了，感觉现在又有点忘了，现在把这个博客写出来，也能够起到一个复习的作用。

因为我感觉我目前对BFC并不是特别清楚，所以在写博客的过程中我也会查资料，所以我把本文题目设为“BFC解惑”。

我今天要讲的是BFC，那么什么是BFC，直接来看看MDN里关于BFC的描述：

块格式化上下文（Block Formatting Context，BFC） 是Web页面的可视化CSS渲染的一部分，是布局过程中生成块级盒子的区域，也是浮动元素与其他元素的交互限定区域。

这一句话讲得很官方，如果事先没有对BFC有什么了解的话应该很难搞明白这句话的含义。MDN里并没有对BFC进行一个详细定义，可以这样认为，BFC是一个特征，具有这个特征的元素可以通过这个特征实现一些功能。那么就从BFC的功能入手，来探讨BFC到底是怎么一回事。

# 创建BFC

如何创建BFC？虽然MDN里一开始没有讲明白BFC到底是什么，但是如何创建BFC是讲得很清楚的。创建了BFC的元素中的所有内容都会被包含到该BFC中。下列方式会创建BFC：

  • 根元素或包含根元素的元素
  • 浮动元素（元素的 float 不是 none）
  • 绝对定位元素（元素的 position 为 absolute 或 fixed）
  • 行内块元素（元素的 display 为 inline-block）
  • 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
  • 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
  • overflow 值不为 visible（可见） 的块元素
  • display 值为 flow-root 的元素
  • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）
  • 等等

其中要注意display:flow-root这条属性仅仅只有让当前元素触发BFC的功能。

以上列出了非常多的项，还有等等没有列出来，满足上面条件的任何一条就会为它们的内容建立一个BFC。BFC的全称是“块格式化上下文”，之前博客讲过堆叠上下文，那个时候也列出了许多使元素获得堆叠上下文的条件。所以就可以进行一个类比，堆叠上下文是元素在垂直屏幕方向上的布局基准，BFC或许就是Block元素的一种布局基准。

# BFC的功能

前面说了那么多BFC，到底BFC有什么用还是不知道。这一节探讨的是BFC的功能，在上课的时候也是直接对BFC的功能进行讲解的，并没有把BFC的布局规则先讲清楚，先看看别人总结归纳的BFC布局规则：

  1. 内部的Box会在垂直方向，一个接一个地放置。
  2. Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  3. 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  4. BFC的区域不会与float box重叠。
  5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  6. 计算BFC的高度时，浮动元素也参与计算

这些归纳很好，BFC的特性也一目了然了，但是这些特性能够在布局方面带来什么便利呢？接下来通过几个例子，把上面的几个特性每一条都通俗地归纳一下。

# 1、包裹浮动元素

具有BFC的父元素可以包裹住内部的浮动的子元素。这是利用了“创建了BFC的元素中的所有内容都会被包含到该BFC中”这条特性。BFC会包裹住BFC中所有内容，即是子元素是浮动元素。下面来看一个例子：

<div class ="father" style="border:1px solid cyan;width:200px;"> 
   <div class ="son" style="float:left;border:1px solid red;width:100px;height:100px;">
   </div>
</div>

此时父元素“father”没有创建BFC，子元素是浮动的，所以包不住，给它加上一条“display:flow-root”属性，使其触发BFC：

<div class ="father" style="border:1px solid cyan;width:200px;display:flow-root"> 
   <div class ="son" style="float:left;border:1px solid red;width:100px;height:100px;">
   </div>
</div>

父元素获得BFC之后立马包裹住了浮动的子元素，因此在计算BFC的高度时，浮动元素也要参与计算。

# 2、margin合并

同一个BFC中，相邻两个块级盒之间的竖直margin会合并，并且内部的Box会在垂直方向，一个接一个地放置。下面是一个例子：

<div  class="wrapper3" style="overflow:hidden;border:1px solid green;width:250px;">
    <div class ="father" style="border:1px solid cyan;width:200px;height:100px;margin:10px;"> 
    </div>
    <div class ="son" style="border:1px solid red;width:100px;height:100px;margin:10px;">
    </div>
</div>

给父元素的两个子元素都设置了10px宽度的margin，显而易见地，两个子元素的margin合并在一起了。这是基于“Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠”而产生的效果。

要想消除这一效果也很简单，只要让这两个子元素不再处于同一BFC内即可，可以给里面的其中一个子元素加上BFC，或者将子元素再用一层容器包裹起来即可：

<div  class="wrapper4" style="overflow:hidden;border:1px solid green;width:250px;">
    <div class ="father" style="border:1px solid cyan;width:200px;height:100px;margin:10px;"> 
    </div>
    <div class="wrapperinner"style="display:flow-root">
        <div class ="son" style="border:1px solid red;width:100px;height:100px;margin:10px;">
        </div>
    </div>
</div>

# 3、自适应两列布局

BFC可以使相邻的两个元素界限分明。可以使用“ float:left + overflow:hidden ”做左右布局,下面是一个例子：

1
2
<div class="div1" style="width:100px;height:150px;float:left;border:1px solid #f66;">1</div>
<div class="div2" style="width:200px;height: 200px;border:1px solid #fcc;">2</div>
1
2

比如以上这种情况，div1是左浮动的，因此具有BFC。因为BFC就是页面上的一个隔离的独立容器，容器里面的元素与外面的元素互不影响。因此div2虽然沉在div下方，但是div2中的文字仍然要与div1做出区隔。

再试着为div2也创建一个BFC，给div2添加一条“overflow:hidden”：

1
2
<div class="aside" style="width:100px;height:150px;float:left;border:1px solid #f66;">1</div>
<div class="main" style="width:200px;height: 200px;border:1px solid #fcc;overflow:hidden;">2</div>
1
2

div2马上与div1划清楚了界限。前文讲到了，BFC的区域不会与float box重叠。因为float box也具有BFC特性，而BFC包裹的元素与外部元素互不干扰。通过使用“ float:left + overflow:hidden ”，成功实现了左右布局的效果。

# 总结

前文举了几个例子，把之前提到的六个BFC的特性都逐条展示了一遍。现在对BFC做一个总结，而且做一个正式的解惑，把一些问题解答一下。

# 文档流与BFC有什么区别？

div的高度时它内部文档流中元素高度的总和决定的。文档流即是是内联元素从左往右，块级元素从上往下排列。文档流影响的是元素的排列顺序。

BFC是div的特性，创建了BFC的元素中的所有内容都会被包含到该BFC中。BFC的高度计算时包括BFC内包含的所有元素。BFC影响的是具有BFC特性的元素的布局基准。

# BFC是必要的吗？

之前提到，BFC可以用来清楚浮动。但是已经知道，使用.clearfix也可以清除浮动，而且比利用BFC特性清除浮动更好，因为不会产生其他可能的副作用。

BFC还可以做左右自适应布局。做左右自适应布局的方法之前也学过，可以使用flex布局，这样比利用BFC特性还要方便。

BFC可以包裹子元素。但是给父元素添加border-top也可以挡住子元素，所以在没有BFC的情况下也可以包裹住子元素。如下图所示，子元素设置了一个向外延伸的50px的margin-top，通过为父元素添加border也可以将父元素内部本来包裹不住的子元素包裹起来：

添加border-top前，父元素与子元素重叠，没有包住margin-top：

<div class ="father" style="background:red;width:100px">
   <div class ="son" style="background:green;height:100px;margin-top:100px;width:100px">
   </div>
</div>

# 如何回答关于BFC的面试

不要试图讲清楚BFC是什么，因为前文讲过，BFC是一个比较难定义的东西，最好通过将例子来回答关于BFC是什么这个问题：

  1. 用BFC可以做清除浮动该做的东西，用BFC可以包住浮动元素。
  2. 用BFC可以使两个相邻元素互不干扰，做左右自适应布局。
