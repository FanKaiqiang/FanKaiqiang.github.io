【分析】JS实现排序算法2——选择排序

广宇 2018-10-8 JavaScript 算法

今天晚上本来想做完Canvas画板的，但是其中因为一些事情耽误了一些时间，而且之前学的怎么做的方法以及一些细节都忘得差不多了。感觉有点惭愧，但是只能明天再做了。但是今天晚上总不能什么也没干，所以今天就继续写一篇关于JS实现算法的博客。现在也天色晚了，就写一篇短的，划划水。

这次就讲一下选择排序，和冒泡排序相似，它也是给数字排序的一种算法。

先来看看百度百科上的定义：

选择排序（Selection sort）是一种简单直观的排序算法。它的工作原理是每一次从待排序的数据元素中选出最小（或最大）的一个元素，存放在序列的起始位置，直到全部待排序的数据元素排完。 选择排序是不稳定的排序方法。

现有一串待排序的数字，需要用选择排序完成排序，如果使用自然语言描述，大概需要以下步骤：

  1. 将数组中第一个数字标记为“最小数”，与第二个数字比较。
  2. 若第一个数字大于第二个数字，则将第二个数字标记为“最小数”
    若第二个数字大于第一个数字，则第一个数字仍为“最小数”。
  3. 将“最小数”与第三个数字比较，得出的较小的数标记为最小数。
  4. 继续向后比较，重复步骤3，经过n-1次比较后得到数组中的最小数（n为数组长度）。
  5. 将最小数与第一个数字交换位置，第一轮排序完成。
  6. 第二轮排序从第二个数字开始，排序完成后数组第二小的数字与第二个数字交换位置。
  7. 经过n-1轮排序，所有小数字都被大数字前方，排序完成。

以下则是通过JS实现选择排序的代码：

function sectionSort(arr){
    for(let min = i = 0; i < arr.length - 1; i++){
        min = i
        for(let j = i + 1; j < arr.length-i-1; j++){
            if(arr[min] > arr[j]){
                min=j
            }
        }
        [ arr[i], arr[min] ] = [ arr[min], arr[i] ]
    }
}

其中：

  1. “for(let min=i=0;i < arr.length-1;i++)”其中i代表排序的轮数，排序需要经过length-1轮的排序。
  2. 同时i也代表第一个排序的数字在数组中的下标值，第一个数此时被标记为最小数，即“min=i=0”。
  3. 其中length为数组长度。每经过一次排序i（排序轮数）加1，i=length-1时退出循环，完成排序。
  4. 每轮排序开始时min=i，即将下标为i的数字标记为最小数。
  5. “for(let j = 0; j < arr.length-1-i; j++)”这一段代表当前被选中比较的数组的下标，每次比较将会在到下标为length-1-i时完成该轮比较，即j=length-i-1时退出循环。
  6. arr[min] > arr[j]时，将min赋值为j，即将j标记为最小数。
  7. 得出该轮排序的最小数后，将最小数与下标为i的数交换位置。

选择排序的时间复杂度为 O(n^2)，数组长度每增加10倍，排序长度增加100倍。

由此可见，虽然在JS的实现上，选择排序的代码看起来比冒泡排序要长一点点，但是起到的效果是相同的。通过冒泡排序，可以渐渐地将较大的数字往右边移动。而通过选择排序，每次都是将最小的数字取出，并放在最前方。这大概就是两种排序在思路上的区别。
