【归纳】移动端页面开发

广宇 2018-10-5 CSS

今天晚上我就继续整理之前学习的知识，这就是我的第十三篇博客。现在正值国庆假期，一整天都在听课，感觉写博客的时间反而不够多了，所以今天晚上就停下来先整理一下之前的知识。今天早上在做我的导航的时候感觉以前的一些知识用的还不够熟练，是值得警惕的。

本篇博客就将移动端页面开发以及动态REM的知识都在这一次博客里归纳起来，希望能起到一个复习的作用。

# 移动端的特点

和平时接触到的PC页面不同，移动端的页面看起来像是一个全新的领域。因为在平时浏览网页的过程中，移动端和PC端的交互区别是非常大的。作为一个正在学习前端知识的人，我就会有这样的疑惑，移动端的页面开发的知识和我之前学习的知识区别大吗？这个时候就要先了解移动端的特点。在学习过程中，我做了如下记录：

  • 手机上没有hover

手机上是没有hover事件的，如果事件是通过hover触发的，在手机上就失效了。

  • 有touch事件

同样地，在PC端没办法实现touch事件，但是在手机上可以办到。

  • 没有resize

手机端不能放大缩小屏幕。手机浏览器的宽度永远等于屏幕宽度，这也是符合平时浏览习惯的。

  • 没有滚动条

在浏览移动端页面时，下滑的操作一般通过滑动手机屏幕完成，在滑动的时候屏幕边上一般才会出现一个指示位置的指示条。

  • 手机上没有IE

不用考虑IE兼容性

大概了解了以上这些信息，大概对移动端有了一个初步的了解。接下了我再来归纳一些关于移动端开发的知识。

# 响应式页面

什么是响应式页面？响应式网站设计是一种网络页面设计布局，其理念是：集中创建页面的图片排版大小，可以智能地根据用户行为以及使用的设备环境进行相对应的布局。就是说，我改变我浏览器的宽度，页面的样式也要随之变化。这其中就要学习媒体查询的知识了。

媒体查询的概念 那什么是媒体查询呢，先来看看MDN上的概念：

一个媒体查询由一个可选的媒体类型和零个或多个使用媒体功能的限制了样式表范围的表达式组成，例如宽度、高度和颜色。媒体查询，添加自CSS3，允许内容的呈现针对一个特定范围的输出设备而进行裁剪，而不必改变内容本身。

大概就是这样的意思。可以了解，通过媒体查询设定以页面的宽度为判断条件，这样写出来的样式就可以限定在一定的页面宽度下使用了。下面是一个例子：

<style>
    @media(max-width:800px){
        body{
            background:red;
        }
    }
</style>

媒体查询的优先级 这个例子是在页面宽度在800px以下时，body的背景显示为红色。除此之外，还可以写一些其他的样式，只要满足这个判断条件的媒体，就会触发这个样式的使用。

<style>
    @media(max-width:800px){

            background:red;
        }
    }

像这种情况，如果同时满足两个条件，则后写的条件优先级高。所以此时若页面宽度小于六百，body的背景将会显示为绿色。如果设计的是手机端页面，一般会设置max-width=451px。

在link标签中建立媒体查询 上面提到的媒体查询都是直接在html中直接使用style标签来引入媒体查询的样式。如果是通过引入外部文件中的样式，可以这样写：

<link rel="stylesheet" href="./style.css" media="(max-width=320px)">
1

这样的话，只有在满足媒体查询的条件时CSS才会生效，但是浏览器仍然会下载这个CSS。

使用JS改变元素的class 在不同屏幕的两种状态会选择媒体查询，如果是在相同屏幕下的两种状态一般会通过JS来实现，比如：

yy.classList.toggle('active')
1

可以用这个方法实现给yy元素添加一个类“active”，如果已经存在一个名为“active”的类，就删去这个类。通过这个方法可以结合媒体查询改变元素的class，从而实现在同一个屏幕下使元素呈现不同的状态。

解除手机端的默认缩放 在平时手机上网的时候也会发现，如果使用手机端打开一个电脑网页，可以发现，手机端显示电脑网页会模拟屏幕宽度为980px的效果，这个效果可以帮助在移动端上访问电脑网页。

但是如果有这个缩放存在，是会影响响应式页面的开发的，因为手机端在此情况下会模拟自己的浏览器宽度为980px，这样不管媒体查询怎样设定宽度都是没用的。所以要在移动端开发前，添加如下的代码：

<meta name="viewport" content="maximum-scale=1.0,minimum-scale=1.0,user-scalable=0,width=device-width,initial-scale=1.0"/>
1

通过这个<meta>标签，将浏览器的宽度设定为设备宽度，并且禁用了缩放。这个标签的快捷键为“meta:vp”。

不过，响应式页面并不实用。在很多时候并不经常见到响应式页面，比如淘宝之类的大部分网站会选择为移动端单独开发一个界面，即做两个站点或者两个HTML。

# 动态REM

动态REM是手机专用的自适应方案。那么什么是REM呢？接下来就先从CSS中的长度单位讲起。

（虽然本图也是rem，但是图文无关）

CSS长度单位 平时主要能够见到如下单位：

  • px：像素
  • em：字长，当前元素的font-size；
  • vh：视口高度，100vh即代表占满整个屏幕；
  • vw：视口宽度
  • rem：根元素（html元素）的font-size；

这个font-size属性是平时设定字体的大小的时候用到的。这和手机端做自适应有什么关系呢？

手机端的自适应 要做手机端的自适应，就是要这个页面到不同的页面中会自动根据这个浏览器的宽度来呈现效果。如果不使用之前所提到的媒体查询，在页面宽度产生变化的时候，元素宽度要怎样才能随之变化呢？

问题的关键在于，元素的高度没法与宽度产生配合。因为页面宽度是固定的，但是高度不是固定的，二者没有关联。为了这些元素能够等比例放大，这时候就要用到rem了。

使用rem就是为了解决宽高之间没有关联的问题。因为页面高度时不固定的，所以一切都要以宽度为基准。其实也可以使用vw这个单位来给元素设置宽高，但是这个单位兼容性差。下面就是使用JS将rem与页面宽度产生关联的方法。

<script>
    var pageWidth = Window.innerWidth
    document.write('<style>html{font-size:'+ pageWidth +'px;}</style>')
</script>

通过doument.write()方法可以往HTML里添加内容，上面的代码设定了html的font-size等于页面宽度，即1rem=100vw。这样宽高就产生联系了。

12px原则 像前文一样设定的话在使用rem的时候会出现小数，比如 width = 0.4 rem等等，所以可以在设定的时候进行优化，比如讲pageWidth除以100，这样就可以使用整数的rem了。

但是这样会出现一个问题，网页的默认font-size是16px；在Chrome中，网页的默认最小字号为12px。如果pageWidth除以100，如果1rem小于12px，那么1rem就会默认为12px。所以要注意避免这种情况的出现，或者去更改Chrome的设置，这样也能解决这个问题。

同时也要注意，浏览器无法显示1px一下的像素，如果元素特别小，这时候使用rem就不是那么方便了，rem是可以和其他元素同时使用的。
