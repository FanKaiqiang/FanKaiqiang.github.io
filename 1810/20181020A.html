【归纳】IFC解惑

广宇 2018-10-21 CSS

今天坐飞机回宁波了，算是我自己正式地终结了我自己的秋招历程。事实上我基本没有参加，所以我就干脆提前回来过几天一个人的生活。一个人呆在宿舍，结合自己的处境，越发感到孤寂。倒不如先写篇博客压压惊，今天稍微学习晚一点也没什么关系，反正明天也是周末，争取这个周末正式完结CSS深入浅出。

这次的主题是IFC，之前讲过BFC，块格式化上下文。这回讲的IFC是与BFC对应的“内联格式化上下文”，按说之前写过了BFC的博客，这回写IFC应该也会得心应手，没想到我上网一查，IFC的资料和BFC相比实在是太少了，几乎很难找到，但也不是没有。所以我这会就老老实实地复盘一下课上的内容。因为查不到资料，所以很多地方都是疑惑，所以这次对IFC也是要解惑的。

因为手上没有资料，一上来也讲不了概念，只能想象。与BFC类似地，满足某些条件的元素便会具有IFC的特性，IFC则是内联元素的布局基准。那就从内联元素的几个比较具有特点的属性开始讲解，以从外向内的方法了解IFC的特征。

# 什么是font-size?

font-size是经常用到的一种属性。如果要问到font-size是什么，我想第一反应肯定是字体大小，可是font-size真的就是字体大小吗？可能很多人以为我说了一句废话，就拿博客中的文字举例。

段落中的文字都是一个字号的，比如“你”、“M”，这两个字的font-size都是相等的。但是font-size指的是“你”的高度还是“M”的高度呢？所以说，font-size与字体大小可能没有直接关系。

之前讲到过一个长度单位，em。之前讲REM的时候讲到过，em就是当前字体的尺寸，1em的大小就等于当前的font-size的大小。实际上font-size就只是确定了这个而已，那么em又是什么呢？这就和字体的原理有关了。

一款字体会定义一个 em-square，它是用来盛放字符的容器。这个 em-square 一般被设定为宽高均为 1000 相对单位。字体度量都是基于这个相对单位设置的，包括 ascender（顶部高度）、descender（底部高度）、capital height（大写字高度）、x-height（小写字高度） 等。

所以说font-size是基于em-square设置的，不同的字体的em-square不同，所以即使设置相同的font-size也会得出不同的字体高度，font-size与字体大小没有确实直接关系。

# line-box是什么？

顾名思义，line-box就是“行盒”。那什么是行盒呢？

当 p 元素出现在屏幕上时，它可能包含了多行内容，每行内容由多个内联元素组成（内联标签或者是包含文本的匿名内联元素），每一行都叫做一个 line-box。

这个line-box就我个人感觉和BFC的一点有些相似。被BFC包裹的子元素会被BFC影响布局，在line-box中的子元素的布局方式也应该要按照IFC的规则来。

# line-height又在指定什么？

刚才在font-size讲完之后插了一些关于行盒的叙述，有点突兀，实际上是要为这一段做准备。line-height就字面意思上可以理解为“行高”，那么事实上line-height到底在指定什么呢？

line-height是由设计师内置的字体推荐行高决定的。line-height决定的是内联的真实占位高度。在没设定行高时，line-height默认为行盒高度，因此默认下的line-height是不确定的。

下面看一个例子，不同两种的字体都设定为100px的line-height时，包裹这两种字体的父元素的高度为何会大于100px？如果是使用网页看这篇博客的话，可以直接打开开发者工具查看p元素的高度。按说两个子元素行高都是100px，父元素的行高为什么不是100px？

# vertical-align是什么？

vertical-align也是计算line-box的高度的重要因素.甚至可以说 vertical-align 是内联格式化上下文（IFC）中最重要的属性。

vertical-align可以看成决定是行盒内元素与行盒的对齐方式的属性。它的默认值为baseline，即是基线对齐。除此之外，它还有其他几种值:

  • vertical-align: top / bottom，表示与 line-box 的顶部或底部对齐
  • vertical-align: text-top / text-bottom，表示与 content-area 的顶部或底部对齐
  • vertical-align: middle，表示父元素baseline的高度加上度元素x-height的一半来对齐当前元素垂直方向的中点。

但是因为 baseline 所处的高度跟字体有关，x-height 的高度也跟字体有关，所以 middle 做字体的对齐也是不靠谱的。内联元素的对齐受太多因素影响，不同的字体难以对齐。

# 图片下面为什么有空隙

在之前做个人简历的时候，我遇到过这种问题，包裹图片的div总是会与图片在底部形成一条缝隙，不能完全重合。当时不知道是为什么，现在这个问题总算是弄明白了。

使用div包裹img元素时，在img底部会有一些空隙。这是为什么呢？可以知道，img也是内联元素，内联元素是与基线对齐的，虽然div中并没有可以与img对齐的内联元素，但是img仍然会与div中的默认字体的baseline对齐。而img的基线即是img的底部。因此会出现图片下产生空隙的情况。

通过修改对齐方式可以解决这个问题，只要图片不是与div的基线对齐就可以：

img{
    vertical-align:top;
}

还可以将img设为display:block，这样img就不是内联元素，没有IFC的特性了。但这样可能引起bug，并不推荐.

# 总结

在接触IFC的时候可能会遇到两个bug：

  1. 内联元素难以对齐，用CSS难以完全对齐不同的字体，而用vertical-align对齐不靠谱，只能凭运气。做居中布局使用flex，不要使用vertical-align。只有用vertical-align消除图片下方空隙才是靠谱的。
  2. 两个内联元素之间有空格，可以通过给父元素加flex或者给子元素加float使之浮动的方法解决。

最后借用一下老师的一个总结

  • IFC 真的很难懂
  • 所有的字体都有两个高度，font-size与line-height
  • line-height: normal 是基于字体度量计算出来的
  • line-height: n (n=1,2,3…) 可能得出一个比 line-box 还要矮的 line-height
  • vertical-align 不靠谱
  • line-box 的高度的受其子元素的 line-height 和 vertical-align 的影响
  • 无法轻易的用 CSS 来控制字体度量
