【记录】JS 里的数据类型转换

广宇 2018-10-14 JavaScript

今天把JS的类型这节课学掉了，课程里讲了很多，这里就做一个记录。因为内容很长，所以我可能会分成两次来写。一来这样写起来没有那么累，二来看起来也没有那么累。

今天晚上冲个刺，就把课上关于JS数据类型的转换的内容归纳一下。

在上篇博客提到，JavaScript里有七种数据类型：

  • 数值（number）
  • 字符串（string）
  • 布尔（boolean）
  • 对象（object）
  • null
  • undefined
  • Symbol

但是在实际运用中可以发现，许多数据的数据类型在某些情况下是会发生转换的。那么这些数据类型之间的转换方法就是这次博客所要讲述的内容。其中的重点在与字符串、数字、布尔这三种数据类型的转换上，Symbol这种数据类型暂时不做讨论。

# 其他类型转为字符串

# 1、toString()

toString() 方法可把一个逻辑值转换为字符串，并返回结果。下面直接来看各种数据类型转换的实例：

数值：

var n = 1
n.toString()  //   "1"
1
2

布尔：

var b = true
b.toString()  //   "true"
1
2

但是要注意null与undefined是不能通过这种方法转换为字符串的：

var e1 = null
e1.toString()  //   Cannot read 'toString'of null
var e2 = undefined
e2.toString()  //   Cannot read 'toString'of undefined
1
2
3
4

原因是null与undefined中并没有toString()这个API。

Object使用toString()方法进行类型转换并不能得到想要的结果：

var object = {name:'frank'}
object.toString()   //   "[object,Object]"
1
2

可以发现，虽然object使用toString()方法没有报错，但是它输出的字符串并不是object本身的内容。所以要想通过这种方法将内容输出为字符串是不可行的。

此外toString()方法的原理在许多地方都得到使用。比如经常见到的console.log()，理论上console.log()只能接受字符串，在输出内容不为字符串时，会先将内容使用toString()方法进行类型转换。

同样地，在引用对象中的值的时候，如obj[key]。在key不为字符串时，也会调用toString() API 将其转化为字符串。

# 2、转化为String的简便方法

和toString()方法相比，还有一种更为简便的方法，即“加空字符串”法：

1 + ''         //   "1"
true + ''      //   "true"
object + ''    //   "[object,Object]"
null + ''      //   "null"
undefined + '' //   "undefined"
1
2
3
4
5

这种方法实现的方法很简单，就是将要转化的数据“加上”一个空字符串。从上面的例子可以看出，在对数值、布尔、对象这三种数据类型的转换中，这种方法的表现是与toString()方法相当的。不仅如此，这种方法还可以将null和undefined转为字符串，这是优于toString()方法的地方。

这种方法的原理是，当“+”号发现左右两边有一边是字符串时，它就会将另一边也转换为字符串。它会希望得到一个字符串的结果。因此会出现一个需要注意的情况：

1 + '1' //   "11"
1

这种结果与常理相悖，事实上也在平时也不会有人这么做。但是需要了解这种情况，即数值与字符串相加并不会得到数字的结果，而是连接起来的字符串。

这种与空字符串相加的方法比toString更为常用。除此之外，String（全局方法）也可以将其他类型转为字符串，这种方法和“加空字符串”法一样强大：

var n = 1
window.String(n)             //   "1"
window.String(true)          //   "true"
window.String(object)        //   "[object,Object]"
window.String(null)          //   "null"
window.String(undefined)     //   "undefined"
1

# 其他类型转为布尔值

# 1、全局函数Boolean

通过Boolean函数可以将其他类型转为布尔值：

Boolean(1)            //   true
Boolean(2)            //   true
Boolean(0)            //   false
Boolean('')           //   false
Boolean(' ')          //   true
Boolean(null)         //   false
Boolean(undefined)    //   false
Boolean({})           //   true
1
2
3
4
5
6
7
8

可以发现，在某些特殊情况下Boolean()函数会将数据转为false。比如0、空字符串、null、undefined等。但是空格字符串与空对象得到的结果是true。

# 2、“!!"取反再取反

通过这种方法可以更为方便地实现布尔值的转换，并且得到的效果和Boolean()函数是一致的:

! true      //   false
!! true     //   true
!! 0        //   false
!! "abc"    //   true
1
2
3
4

可以看出，这种方法的实现是依赖了“!"的取反功能，!可以将数据取为相反的布尔值，如果发现不是布尔值就会将其进行转换。一次取反后再次取反，即“!!”，就可以得到需要的原本的布尔值了。

可以在此进行一次归纳，JS中有5种特殊值是会转换为false的：

  • number：0 、NaN
  • string：''（空字符串）
  • null
  • undefined

这五种值也被成为falsy值，除了这几种值之外，其他所有数据都会被转换为true。值得一提的是，所有object都会被转换为true，包括空对象。

# 其他类型转换为数值

数字字符串即是包含数字的字符串。将其转换为数值的方法有很多种：

# 1、Number()函数

Number() 函数把对象的值转换为数字。如果参数是 Date 对象，Number() 返回从 1970 年 1 月 1 日至今的毫秒数。如果对象的值无法转换为数字，那么 Number() 函数返回 NaN：

var n = new Date()
Number("1")          //   1
Number("1a")         //   NaN
Number(true)         //   true
Number(false)        //   false
Number(n)            //   1539443280279
1
2
3
4

# 2、parseInt()函数

parseInt() 函数可解析一个字符串，并返回一个整数。parseInt函数的格式一般如下：

parseInt(string, radix)

其中string为要解析的字符串，radix为解析出来的数字为多少进制。下面是几个使用示例：

parseInt("10");			//   10
parseInt("19",10);		//   19
parseInt("11",2);		//   3 
parseInt("17",8);		//   15 
parseInt("1f",16);		//   31 
parseInt("a");			//   NaN
parseInt("12a");		//   12
parseInt("010");		//   10（待定）
1
2
3
4
5
6
7
8

parseInt是从头开始解析，直到遇到不可解析的字符才返回结果。

需要注意的是，在数字字符串以零开头且没有指定基数的情况下,结果是不确定的。那么具体是哪个基数由实现环境决定。ECMAScript 5 规定使用10，是因为如果radix没有指定或者为0，参数会被假定以10为基数来解析，且字符串中开头的0会被忽略。比如在Chrome下就是如此。但是并不是所有的浏览器都遵循这个规定。因此，永远都要明确给出radix参数的值。

# 3、parseFloat()函数

parseFloat() 函数可解析一个字符串，并返回一个浮点数：

parseFloat('3.14')        //3.14
parseFloat('abc')         //NaN
parseFloat('3.14abc')     //3.14
parseFloat('abc3.14')     //NaN
1
2
3
4

可见，该函数会检查字符串中的首个字符是否是数字。如果是，则对字符串进行解析，直到到达数字的末端为止，然后以数字返回该数字，而不是作为字符串。

# 4、减零

将字符串“减”去零，这种方法既有parseInt()的部分功能，又有parseFloat()的部分功能：

'1' - 0         //  1
'1.23' - 0      //  1.23
'1.23aa' - 0    //  NaN
1
2
3

# 5、取正

将字符串“取正”，这种方法的效果和减零的方法相似：

+ '1'         //  1
+ '1.23'      //  1.23
+ '1.23aa'    //  NaN
1
2
3

其中，“+”意为取其原本的值，但是是以数字的形式。注意这种方法并不是取绝对值。同理，“-”即可以取其相反数。
