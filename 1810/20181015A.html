【归纳】Bootstrap的使用方法

广宇 2018-10-15 CSS Bootstrap

之前把CSS深入浅出的课程学完了，这段时间都在学习JS有关的内容，CSS 有一段时间没有学了，之前的笔记再不整理就要忘了。在正式把CSS深入浅出这节课完成之前，我将会把这门课程讲到的每个重要的知识点都做一次归纳。这次的主题是前一段时间学的Bootstrap的使用方法。

Bootstrap是美国Twitter公司的设计师Mark Otto和Jacob Thornton合作基于HTML、CSS、JavaScript 开发的简洁、直观、强悍的前端开发框架，使得 Web 开发更加快捷。

Bootstrap的最初目的是让后端只需要通过拷代码就能够搞出页面，让前端开发更快速、简单。所有开发者都能快速上手、所有设备都可以适配、所有项目都适用。全球数以百万计的网站都是基于 Bootstrap 构建的。可见Bootstrap是颇受欢迎的。

事实上，我当前实习的公司也是会使用Bootstrap的。虽然作为前端开发可能在以后的工作中并不会用到Bootstrap，但是Bootstrap的思想是所需要学习的。

# 使用Bootstrap布局

使用Bootstrap做页面，首先要考虑页面布局的问题。那么如何使用Bootstrap的页面布局？这时候就要了解一下Bootstrap的栅格系统。

在中文文档中是这样介绍栅格系统的：

Bootstrap 提供了一套响应式、移动设备优先的流式栅格系统，随着屏幕或视口（viewport）尺寸的增加，系统会自动分为最多12列。它包含了易于使用的预定义类，还有强大的mixin 用于生成更具语义的布局。

# Bootstrap的响应式

刚才大致地讲了一下Bootstrap的布局方式，Bootstrap的布局是使用由div组成的栅格系统完成的。为什么要使用div而不是其他标签？因为div没有默认样式，使用其他标签的话要消除标签默认样式的影响。

讲完布局，就要讲页面的响应式。如何使页面在不同的设备上都能够以所希望的方式显示？Bootstrap默认有四种媒体查询的分界值，通过下表可以详细查看 Bootstrap 的栅格系统是如何在多种屏幕设备上工作的：

  • .col-xs-：超小屏幕 手机 (<768px)
  • .col-sm-：小屏幕 平板 (≥768px)
  • .col-md-：中等屏幕 桌面显示器 (≥992px)
  • .col-lg-：大屏幕 大桌面显示器 (≥1200px)

在之前的例子使用的全都是在中等屏幕下显示的情况。如col-xs-6即代表在xs条件下div占据六格。如下方代码，可以通过给div添加不同的类来修改在不同的屏幕下显示的宽度：

<div class="row">
  <div class="col-xs-12 col-sm-6 col-md-8">.col-xs-12 .col-sm-6 .col-md-8</div>
  <div class="col-xs-6 col-md-4">.col-xs-6 .col-md-4</div>
</div>

在没有设定符合条件类时，Bootstrap会优先使用小型屏幕的布局。比如上方代码中row包含的第二个div，在屏幕大小为sm时，会优先使用xs屏幕的布局，而不是md。

使用hidden类可以使元素在不同的屏幕宽度下隐藏。如.hidden-xs可以使元素在小屏幕的情况下隐藏。除此之外，还有.hidden-sm、.hidden-md、.hidden-lg等。通过这个类可以帮助更好地完成响应式布局。

在代码中，每个row都会被分成十二份宽度。row的总宽度并非一定要等于屏幕宽度。row中可以嵌套row，里面的row也会被分为12等分。

Bootstrap 是移动设备优先的。在 Bootstrap 3 中，针对移动设备的样式融合进了框架的每个角落，为了确保适当的绘制和触屏缩放，需要在 <head> 之中添加 viewport 元数据标签:

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">

# 组件

在完成响应式与布局之后，就要在页面中添加一些需要的组件了。Bootstrap的组件添加也是很简单的。作为一个想让前端开发更快速、简单，并且开发者都能快速上手的框架，想要引入组件可以直接进入Bootstrap3中文文档中的“全局CSS样式”以及“组件”模块，里面提供了许多全局CSS样式以及其他复杂的组件：

比如我要做一个下拉的菜单项，就可以直接找到中文文档中“组件”部分关于下拉菜单项的代码，把它复制到之前布局完成的模块里：

/*代码如下*/
<div class="dropdown">
  <button class="btn btn-default dropdown-toggle" type="button" id="dropdownMenu1" data-toggle="dropdown" aria-haspopup="true" aria-expanded="true">
    Dropdown
    <span class="caret"></span>
  </button>
  <ul class="dropdown-menu" aria-labelledby="dropdownMenu1">
    <li><a href="#">Action</a></li>
    <li><a href="#">Another action</a></li>
    <li><a href="#">Something else here</a></li>
    <li role="separator" class="divider"></li>
    <li><a href="#">Separated link</a></li>
  </ul>
</div>

jQuery 插件为 Bootstrap 的组件赋予了“生命”。可以简单地一次性引入所有插件，或者逐个引入到你的页面中。可以在“JavaScript插件”里引入JS组件，做一些CSS组件做不到的事情。

Bootstrap还有定制功能。在中文文档的“定制”模块里，可以根据自己样式的需求，量身定做一套Bootstrap组件并下载使用。除此之外，也可以自己在网上查找Bootstrap的主题，直接引入到自己的项目当中：

感觉这篇博客关于组件的部分我直接一笔带过了，事实上Bootstrap就是那么简单上手。使用Bootstrap可以简单地布局、简单地写样式。通过复制黏贴文档里大家准备好的代码，可以完成很多需求。利用Bootstrap框架，可以省去开发人员许多工作以及学习的时间，就能做出一个满足要求的页面。

要想学好Bootstrap，首先还是要学好CSS，因为无论是布局还是组件，都是围绕着Bootstrap中已经写好的CSS展开的。同时还要再学些JQuery，然后再多读文档，文档中已经很详尽地讲解了关于Bootstrap的各种知识点。如果想要看看其他人使用Bootstrap框架做出来的作品，在文档中的“网站实例”模块，也就是Bootstrap优站精选。在这里有大量别人的优秀作品，可以在这里学习到他们的使用技巧。
