【归纳】grid布局

广宇 2018-12-7 CSS

Grid布局算是我比较早就想学习的一个内容。grid布局是一种全新的布局，CSS深入浅出这个课讲完的时候grid布局还是一种比较新颖的方式，所以老师只是稍微提到了这个强大的布局，并没有细讲。

不过那个时候我就在心中有好好了解grid布局的想法，想着什么时候一定要把这个神秘内容给学学。

凭借这次11月份签到30天的奖励，我直接就换了一节关于grid布局的课程。这是若愚老师以前的直播课，以前就有买下这节课的想法，课程到手之后今天也花了一早上的时间把内容给看完了，可以说是非常划算的。老师课前专门翻译了一篇关于grid布局的文章，里面其实讲得就已经很详细，再结合老师的讲解，我在看完之后就感觉这个知识我已经了解了个大概了，所以今天干脆就把关于grid布局的知识点给归纳一下。

全文大概就是把我自己的笔记和老师的文章大概过一遍，总的来说感觉不是很难，希望能够起到一个巩固知识的效果吧。

# grid布局的基本概念

CSS 网格布局(Grid Layout) 是CSS中最强大的布局系统。 这是一个二维系统，这意味着它可以同时处理列和行，不像 flexbox 那样主要是一维系统。 你可以通过将CSS规则应用于父元素（成为网格容器）和该元素的子元素（网格元素），来使用网格布局。所谓网格布局就是现在页面上画好格子，再讲元素放到画好的格子上，这也就是grid布局的思路。

截至目前，许多浏览器都提供了原生的、不加前缀的对CSS Grid的支持，比如 Chrome（包括Android），Firefox，Safari（包括iOS）和Opera。在目前的项目中，实际上已经可以大胆地使用grid布局，已经完全到了应该学习的时候了。

一开始你需要使用display：grid把容器元素定义为一个网格，使用grid-template-columns和grid-template-rows设置列和行大小，然后使用grid-column 和 grid-row把它的子元素放入网格。在深入了解网格的概念之前，理解术语是很重要的。这里大概地把关于grid布局的所有术语讲解一下：

  • grid item：Grid 容器的孩子（直接子元素）。
  • grid line：组成网格结构的网格线，它们既可以是垂直的（“column grid lines”），也可以是水平的（“row grid lines”），并位于行或列的任一侧。
  • grid track：两个相邻网格线之间的空间。 网格的一列或一行。
  • grid cell：两个相邻的行和两个相邻的列网格线之间的空间。网格的一个格子（网格单元）。
  • grid area：四个网格线包围的总空间。 网格区域可以由任意数量的网格单元组成。

了解了这些基本术语，就可以开始讲讲与这些术语紧密相关的属性，可以正式开始学习grid布局的使用方法了。

# Grid容器（grid container）的属性

# 1、display

将元素定义为 grid contaienr，并为其内容建立新的网格格式化上下文(grid formatting context)。

.container {
  display: grid | inline-grid | subgrid;
}

2
3

其中，inline-grid可以生成一个行级(inline-level)网格。如果你的 grid container 本身就是一个 grid item（即,嵌套网格），你就可以使用 subgrid 来表示你想从它的父节点获取它的行/列的大小，而不是指定它自己的大小。

# 2、grid-template-columns / grid-template-rows

这两个属性被用来定义网格的列和行。通俗地讲，这个属性的作用就是给这个grid容器画网格线。下面是一个例子：

.container {
  grid-template-columns: 40px 50px auto 50px 40px;
  grid-template-rows: 25% 100px auto;
}

# 3、grid-template-areas

借助这个属性，可以把子元素填充到grid容器里。在此之前需要给每个子元素添加grid-area属性，给每个子元素取名字。然后再将这些元素的名字通过grid-template-areas属性填充到对应的格子里。下面是一个例子：

.item-a {
  grid-area: header;
}
.item-b {
  grid-area: main;
}
.item-c {
  grid-area: sidebar;
}
.item-d {
  grid-area: footer;
}

.container {
  grid-template-columns: 50px 50px 50px 50px;
  grid-template-rows: auto;
  grid-template-areas: 
    "header header header header"
    "main main . sidebar"
    "footer footer footer footer";
}

需要注意的是你不是在用这个语法命名网格线，而是在命名区域。 当你使用这种语法时，区域两端的网格线实际上是自动命名的。 比如，如果网格区域的名称是foo，那么区域的起始的行网格线和列网格线名称是 foo-start，并且区域终点的行网格线和列网格线名称是 foo-end。 这意味着某些网格线可能有多个名称，比如上面的例子中最左边的一条网格线有三个名字：header-start，main-start 和 footer-start。

# 4、grid-template

这个属性是上面 grid-template-rows、grid-template-columns、grid-template-areas 这几个属性的的简写。它也可以使用一个更复杂但相当方便的语法来指定这三个值。下面是一个例子：

.container {
  grid-template:
    [row1-start] "header header header" 25px [row1-end]
    [row2-start] "footer footer footer" 25px [row2-end]
    / auto 50px auto;
}
/* 以上等价于： */
.container {
  grid-template-rows: [row1-start] 25px [row1-end row2-start] 25px [row2-end];
  grid-template-columns: auto 50px auto;
  grid-template-areas: 
    "header header header" 
    "footer footer footer";
}

# 5、grid-column-gap / grid-row-gap

这个属性用于定义网格线的间隙。需要注意的是，网格的边线并不会与grid容器的边框产生间距。这两个属性可以使用grid-gap的简写方式。下面是一个例子：

.container {
  grid-template-columns: 100px 50px 100px;
  grid-template-rows: 80px auto 80px; 
  grid-column-gap: 10px;
  grid-row-gap: 15px;
}
/* 等价于 */
.container {
  grid-template: 80px auto 80px / 100px 50px 100px ; 
  grid-gap: 10px 15px;
}

# 6、justify-items / align-items

这两个属性是分别用于色调grid容器中所有的 grid item 的水平/垂直对齐方式的。也可以通过给单个 grid item 设置justify-self属性来达到设定对齐方式的效果。这个属性接受以下几种值：

  • start: 内容与网格区域的左端对齐
  • end: 内容与网格区域的右端对齐
  • center: 内容位于网格区域的中间位置
  • stretch: 内容宽度占据整个网格区域空间(这是默认值)

# 7、justify-content / align-content

有时，网格的总大小可能小于其网格容器的大小。如果你的所有 grid items 都使用像px这样的非弹性单位来设置大小，则可能发生这种情况。上一条内容定义的是放在格子里的元素的对齐方式，这个属性这是用于设定画好的格子的对齐方式。这个属性接受以下几种值：

  • start - 网格与网格容器的左边对齐
  • end - 网格与网格容器的右边对齐
  • center - 网格与网格容器的中间对齐
  • stretch - 调整g rid item 的大小，让宽度填充整个网格容器
  • space-around - 在 grid item 之间设置均等宽度的空白间隙，其外边缘间隙大小为中间空白间隙宽度的一半
  • space-between - 在 grid item 之间设置均等宽度空白间隙，其外边缘无间隙
  • space-evenly - 在每个 grid item 之间设置均等宽度的空白间隙，包括外边缘

# 8、grid-auto-columns / grid-auto-rows

有时会把需要放置到网格的元素放到画好的网格的外面，这时候就会自动生成网格延伸到这个元素的位置。这条属性就是用来设定这些自动生成的网格大小。下面是一个例子：

.container {
  grid-template-columns: 60px 60px;
  grid-template-rows: 90px 90px
}
/* 下面的item-b被放置在画好的网格线外面 */
.item-a {
  grid-column: 1 / 2;
  grid-row: 2 / 3;
}
.item-b {
  grid-column: 5 / 6;
  grid-row: 2 / 3;
}
.container {
  grid-auto-columns: 60px;
}

# 9、grid-auto-flow

如果并没有指定grid容器中的grid item中网格中的放置方式，这时候就会依据这条属性来确认这些grid item的排列方式。这个属性接受以下几种值：

  • row - 告诉自动布局算法依次填充每行，根据需要进行换行
  • column - 告诉自动布局算法依次填充每列，根据需要进行换列
  • dense - 告诉自动布局算法，如果后面出现较小的 grid item，则尝试在网格中填充空洞

# 10、grid

此属性是所有以下属性的简写：grid-template-rows，grid-template-columns，grid-template-areas，grid-auto-rows，grid-auto-columns和grid-auto-flow。 它同时也将 sets grid-column-gap 和 grid-row-gap 设置为它们的初始值，即使它们不能被此属性显示设置。下面是该属性的格式：

.container {
    grid: none | <grid-template-rows> / <grid-template-columns> | <grid-auto-flow> [<grid-auto-rows> [/ <grid-auto-columns>]];
}

# 子元素（Grid Items）的属性

# 1、grid-column-start / grid-column-end / grid-row-start /grid-row-end

使用特定的网格线确定 grid item 在网格内的位置。grid-column-start/grid-row-start 属性表示grid item的网格线的起始位置，grid-column-end/grid-row-end属性表示网格项的网格线的终止位置。如果没有声明 grid-column-end / grid-row-end，默认情况下，该网格项将跨越1个轨道。网格项可以相互重叠，可以使用z-index来控制它们的堆叠顺序。这些属性接受以下的值：

  • <line>: 可以是一个数字来指代相应编号的网格线，也可使用名称指代相应命名的网格线
  • span <number>: 网格项将跨越指定数量的网格轨道
  • span <name>: 网格项将跨越一些轨道，直到碰到指定命名的网格线
  • auto: 自动布局， 或者自动跨越， 或者跨越一个默认的轨道

下面是一个例子，这几条属性确定了该元素在网格中上下左右的范围，从而实现了将元素填充到网格中。此外，这些属性可以简写为grid-column / grid-row：

.item-b {
  grid-column-start: 1;
  grid-column-end: span col4-start;
  grid-row-start: 2;
  grid-row-end: span 2
}
/* 等价于 */
.item-b {
  grid-column: 1 / span col4-start;
  grid-row: 2 / span 2
}

# 2、grid-area

刚才在讲到父容器的grid-template-areas属性时愿提到了这条属性。这个属性用于给子元素取名，然后再将子元素的名字填写到grid-template-areas属性的相应位置中。另外也可以做为 grid-row-start + grid-column-start + grid-row-end + grid-column-end 的简写形式。使用方法大致如下：

.item {
  grid-area: <name> | <row-start> / <column-start> / <row-end> / <column-end>;
}

# 3、justify-self / align-self

这两个属性在前文也提到过，用于设定当前grid item在网格中的水平/垂直方向的对齐方式。这个属性能接受的值如下：

  • start - 将内容对齐到网格区域的顶部
  • end - 将内容对齐到网格区域的底部
  • center - 将内容对齐到网格区域的中间
  • stretch - 填充网格区域的高度 (这是默认值)
