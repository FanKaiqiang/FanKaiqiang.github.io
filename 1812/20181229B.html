【归纳】JS 对象的一些知识

广宇 2018-12-30 ES6 JavaScript

这次博客的内容主要是 ES6 深入浅出中「新版对象」这一课的笔记整理，这些知识点比较散，内容不只是讲对象在 ES6 中的新特性，所以本文的题目也是「JS 对象的一些知识」，写到哪里算哪里吧。

虽然这些知识比较散，却一点也不能说是简单的，如果都能掌握，对 JS 中对象这种数据类型的理解也能更进一步。

# 1、产生空对象的多种方法

常常使用 new Object() 和字面量标记来初始化对象：

var a = new Object()
var b = {}
1
2

但是这样生成的空对象不是完全的空对象，打开这个对象，会发现他还有一条 __proto__ 属性，指向它的原型 Object.prototype。虽然生成的是空对象，但是还是可以使用原型提供的方法和属性。如何生成一个完全的空对象呢？这里就要用到 Object.create() 方法了。

Object.create()方法创建一个新对象，使用现有的对象来提供新创建的对象的 __proto__ 属性。所以如果设置其指向 null，这个对象就是一个没有原型的完全的空对象了。如果设置其指向 Object.prototype，就能创建出与之前 a、b 效果相同的对象了：

var c = Object.create(null)
var d = Object.create(Object.prototype)
1
2

# 2、ES6 中的计算属性名

从 EC6 开始，对象初始化语法开始支持计算属性名。其允许在[]中放入表达式，计算结果可以当做属性名。这种用法和用方括号访问属性非常类似，现在同样的语法也可以用于对象字面值了：

var param = 'size';
var config = {
  [param]: 12,
  ["mobile" + param.charAt(0).toUpperCase() + param.slice(1)]: 4
};

console.log(config); // { size: 12, mobileSize: 4 }

# 3、对属性读写进行操作

在 JavaScript 中，可以使用 getter、setter 方法来实现对属性读写进行自定义操作。getter 方法将对象属性绑定到查询该属性时将被调用的函数；当尝试设置属性时，setter 方法将对象属性绑定到要调用的函数。

var obj = {
  _age = 18,
  get age(){
    return obj._age
  },
  set age(value){
    if(value<100){
      obj._age = value
    }else{
      obj._age = 100
    }
  }
}
console.log(obj.age); // 18
obj.age = 200
console.log(obj.age); // 100

setter 经常和 getter 连用以创建一个伪属性。就上例而言，事实上对象中并没有 age 属性，在读写过程中实际上是在调用 getter 和 setter。虽然可以使用 getter 和 setter 来创建一个伪属性类型，但是不能将 getter 绑定到一个已经具有值的属性。

# 4、变更原型

任何一个对象都有一个 __proto__ 属性，它指向该对象的的原型，通过obj.__proto__ = ... 可以改变一个对象的原型。不过因为__proto__从来没有被包括在EcmaScript语言规范中，所以使用此属性是有争议的，并不鼓励使用它。现在更推荐使用 Object.getPrototypeOf 和 Object.setPrototypeOf 来读写原型：

var reg = /a/;
Object.getPrototypeOf(reg) === RegExp.prototype; // true 

var dict = Object.setPrototypeOf({}, null);   //  编辑原型
1
2

需要注意的是，修改对象原型的行为在每一个 JavaScript 引擎和浏览器中都是一个非常慢且影响性能的操作，使用这种方式来改变和继承属性是对性能影响非常严重的，并且性能消耗的时间也不是简单的花费在修改原型的语句上, 它还会影响到所有来自该原型链上的对象，如果你关心性能，你就不应该在一个对象中修改它的原型。最好的做法是, 使用 Object.create()创建一个新的且可以继承原型的对象。

# 5、给对象插入或修改属性

在定义变量之后，如果想给这个变量修改属性倒是不难，但是怎么给变量添加 getter、setter 方法呢？这个时候就要用到 Object.defineProperty() 方法了。这个方法会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性， 并返回这个对象。使用语法如下：

Object.defineProperty(obj, prop, descriptor)
1

其中，obj 是要在其上定义属性的对象、prop 是要定义或修改的属性的名称、descriptor 是将被定义或修改的属性描述符。方法的返回值为被传递给方法的对象。

通过给属性设置属性描述符可以操作属性的一些特性。对象里目前存在的属性描述符有两种主要形式：数据描述符和存取描述符。数据描述符是一个具有值的属性，该值可能是可写的，也可能不是可写的。存取描述符是由getter-setter函数对描述的属性。描述符可同时具有的键值如下表：

configurable
（可改变）		 enumerable
（可枚举）		 value
（值）		 writable
（可赋值）		 get
（读取回调）		 set
（写入回调）
数据描述符 Yes Yes Yes Yes No No
存取描述符 Yes Yes No No Yes Yes

如果一个描述符不具有value,writable,get 和 set 任意一个关键字，那么它将被认为是一个数据描述符。如果一个描述符同时有(value或writable)和(get或set)关键字，将会产生一个异常。

下面是使用此方法创建属性的例子：

var o = {}; // 创建一个新对象

// 在对象中添加一个属性与数据描述符的示例
Object.defineProperty(o, "a", {
  value : 37,
  writable : true,
  enumerable : true,
  configurable : true
});

// 对象o拥有了属性a，值为37

// 在对象中添加一个属性与存取描述符的示例
var bValue;
Object.defineProperty(o, "b", {
  get : function(){
    return bValue;
  },
  set : function(newValue){
    bValue = newValue;
  },
  enumerable : true,
  configurable : true
});

o.b = 38;

如果属性已经存在，Object.defineProperty()将尝试根据描述符中的值以及对象当前的配置来修改这个属性。如果旧描述符将其configurable 属性设置为false，则该属性被认为是“不可配置的”，并且没有属性可以被改变（除了单向改变 writable 为 false）。当属性不可配置时，不能在数据和访问器属性类型之间切换。

使用 Object.getOwnPropertyDescriptor() 方法返回指定对象上一个自有属性对应的属性描述符。比如：

var o, d;

o = { get foo() { return 17; } };
d = Object.getOwnPropertyDescriptor(o, "foo");
// d {
//   configurable: true,
//   enumerable: true,
//   get: /*the getter function*/,
//   set: undefined
// }

var obj = {};
Object.defineProperties(obj, {
  'property1': {
    value: true,
    writable: true
  },
  'property2': {
    value: 'Hello',
    writable: false
  }
  // etc. etc.
});
