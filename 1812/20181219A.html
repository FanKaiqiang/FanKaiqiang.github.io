【分析】JS实现排序算法5——希尔排序

广宇 2018-12-20 算法 JavaScript

今天突然感觉不知道写啥了，晚上又有直播课，也想看一下。主要是这几天都看毕设课程去了，没学什么新知识，所以今天就再水一篇博客。

之前学习的排序算法也好久没有跟进了，这篇博客就把希尔排序写一下。也是两个多月前学习的内容，我都已经忘记希尔排序是什么了，这次就当算重新学习一下好了。

已经好长一段时间没看过算法方面的内容了，这回直接来看看希尔排序在维基百科上的解释：

希尔排序，也称递减增量排序算法，是插入排序的一种更高效的改进版本。希尔排序是非稳定排序算法。

希尔排序是1959年由 Donald Shell 发明的，第一个突破O(n2)的排序算法，是简单插入排序的改进版。它与插入排序的不同之处在于，它会优先比较距离较远的元素。希尔排序又叫缩小增量排序。希尔排序是基于插入排序的以下两点性质而提出改进方法的：

  • 插入排序在对几乎已经排好序的数据操作时，效率高，即可以达到线性排序的效率
  • 但插入排序一般来说是低效的，因为插入排序每次只能将数据移动一位

希尔排序通过将比较的全部元素分为几个区域来提升插入排序的性能。这样可以让一个元素可以一次性地朝最终位置前进一大步。然后算法再取越来越小的步长进行排序，算法的最后一步就是普通的插入排序，但是到了这步，需排序的数据几乎是已排好的了（此时插入排序较快）。

假设有一个很小的数据在一个已按升序排好序的数组的末端。如果用复杂度为O(n2)的排序（冒泡排序或插入排序），可能会进行n次的比较和交换才能将该数据移至正确位置。而希尔排序会用较大的步长移动数据，所以小数据只需进行少数比较和交换即可到正确位置。

通俗的讲，就是先将数组中相隔一定步长的数字分为一组，每个组分别进行插入排序，一轮排序结束后，缩小步长的值对数字进行分组，然后再对每个组进行插入排序。当步长为1时，算法变为普通插入排序，这一轮排序完成之后，整个排序过程也就完成了。

下面再看一张希尔排序的模拟过程的动图，与刚才用自然语言描述的方法基本一致：

以下就是通过JS实现插入排序的代码：

function shellSort(arr){
  var temp
  var len = arr.length
  for(var gap = len >> 1; gap > 0; gap = gap >> 1){
    for(var i = gap; i < len; i++){
      temp = arr[i]
      for(j = i - gap; j >= 0 && arr[j] > temp; j -= gap){
        arr[j + gap] = arr[j]         //arr[i] = arr[j]
      }
      arr[j + gap] = temp             //arr[j] = arr[i]
    }
    console.log(arr)
  }
}

var arr = [3, 1, 7, 2, 5, 0, 4, 6]
shellSort(arr)
console.log(arr)

这里一口气叠了三层 for 循环，还用到了平时很少见的 >> 运算符，这里慢慢地把这个代码分析一下：

  • 第一层 for：步长 gap 初状态为数组长度的一半（向下取整），每次循环结束后 gap 除以二，gap等于 0 的时候退出循环。
  • 第二层 for：令 i 等于 gap（当前步长），设置一个变量 temp 等于 arr[i] ，每次循环结束后 i 加一，i 小于数组长度时退出循环。
  • 第三层 for：令 j 等于 i 减去 gap，每次循环结束后 j - gap，当j小于零或 arr[j] < arr[i] 时退出循环。

这里的重头戏主要是第二层与第三层 for。假设 i 是当前数的下标，那么 j 就是 i 所在分组中的前一个数的下标。如果 arr[j] > arr[i]，就对调二者的值，然后再继续比较分组中在 i 前面的数字。第三层 for 主要任务是将分组进行插入排序，而第二层 for 的作用就是在将数字一个一个推入分组，当正则数组都完成了这一轮的插入排序之后（i = len），就相应减少步长 gap ，再重复这个操作。

上面的过程有点绕，我也是想了好一会才想明白，特别是对调操作那里，里面的j + gap（其实是 i）和外面的 j + gap（其实是 j）不是相等的。如果画一个流程图之类的可能有所帮助，这里就不画了，有点麻烦。直接看看这段代码在浏览器中的执行效果：

希尔排序的时间复杂度为时间复杂度为 O(nlog2n)，希尔排序没有快速排序算法快 O(n(logn))，因此中等大小规模表现良好，对规模非常大的数据排序不是最优选择。但是比O(n2)复杂度的算法快得多。

希尔排序非常容易实现，算法代码短而简单。此外，希尔算法在最坏的情况下和平均情况下执行效率相差不是很多，但是快速排序在最坏的情况下执行的效率会非常差。所以，几乎任何排序工作在开始时都可以用希尔排序，若在实际使用中证明它不够快，再改成快速排序这样更高级的排序算法。

总而言之，希尔排序算法是直接插入排序算法的一种改进，减少了其复制的次数，速度要快很多。希尔排序是按照不同步长对元素进行插入排序，当刚开始元素很无序的时候，步长最大，所以插入排序的元素个数很少，速度很快；当元素基本有序了，步长很小，插入排序对于有序的序列效率很高。所以，希尔排序的时间复杂度会比o(n2)好一些。
