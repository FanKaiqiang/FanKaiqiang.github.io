# 一、迭代器

处理集合中的每个项是很常见的操作，主要方式有遍历与迭代。遍历的话也算是使用 for 的常规操作了，大家都比较熟悉。至于迭代，虽然这个名词并不算陌生，但是具体的概念我还是不甚清楚，来看看百度百科中关于迭代的说法：

迭代是重复反馈过程的活动，其目的通常是为了逼近所需目标或结果。每一次对过程的重复称为一次“迭代”，而每一次迭代得到的结果会作为下一次迭代的初始值。

JavaScript 中提供了许多迭代集合的方法，从简单的 for 循环到 map() 和 filter()。上面提到的那几个都是数组的方法，那么能不能自定义支持迭代的对象呢？这就要用到迭代器了。

一个迭代器对象 ，知道如何每次访问集合中的一项， 并跟踪该序列中的当前位置。在 JavaScript 中 迭代器是一个对象，它提供了一个next() 方法，用来返回序列中的下一项。这个方法返回包含两个属性：

  • done (boolean)：用以表示此迭代器的迭代是否结束。如果迭代器还可以继续迭代，则为 false。done 属性默认值为 false。
  • value：迭代器返回的任何值。

迭代器对象一旦被创建，就可以反复调用next()。一旦初始化，next()方法可以用来依次访问对象中的键值：

function makeIterator(array){
  var nextIndex = 0;
  return {
    next: function(){
      return nextIndex < array.length ?
        {value: array[nextIndex++], done: false} :
        {done: true};
    }
  };
}

var it = makeIterator(['yo', 'ya']);
console.log(it.next().value); // 'yo'
console.log(it.next().value); // 'ya'
console.log(it.next().done);  // true

# 二、生成器

# 1、生成器示例

虽然自定义的迭代器是一个有用的工具，但由于需要自己去维护其 next 与 done 的值，显得并不方便。Generators 提供了一个强大的选择：它允许你定义一个包含自有迭代算法的函数，同时它可以自动维护自己的状态。Generator 语法可以被看成是一种生成迭代器的语法糖，Generator Function也被称为生成器函数，它将返回一个 Generator 对象，这个对象就是一个迭代器。

Generator Function 是一个可以作为迭代器工厂的特殊函数。当它被执行时会返回一个新的Generator对象。 如果使用 function* 语法定义函数，则函数将变为 Generator Function。来看一个例子：

function* idMaker() {
  var index = 0;
  while(true)
    yield index++;
}

var gen = idMaker();

console.log(gen.next().value); // 0
console.log(gen.next().value); // 1
console.log(gen.next().value); // 2
// ...

# 2、Generator 对象（迭代器对象）

上面的 Generator 对象的 next() 方法可以接收参数，如果传入了参数，那么这个参数会作为上一条执行的 yield 语句的返回值，例如：

function *createIterator() {
    let first = yield 1;
    let second = yield first + 2; // 4 + 2 
    // first = 4 是next(4)将参数赋给上一条的
    yield second + 3;             // 5 + 3
}

let iterator = createIterator();

console.log(iterator.next());    // "{ value: 1, done: false }"
console.log(iterator.next(4));   // "{ value: 6, done: false }"
console.log(iterator.next(5));   // "{ value: 8, done: false }"
console.log(iterator.next());    // "{ value: undefined, done: true }"

# 3、Generator Function（生成器函数）

调用一个生成器函数并不会马上执行它里面的语句，而是返回一个这个生成器的 迭代器 （iterator ）对象。当这个迭代器的 next() 方法被首次（后续）调用时，其内的语句会执行到第一个（后续）出现 yield 的位置为止，yield 后紧跟迭代器要返回的值。或者如果用的是 yield*（多了个星号），则表示将执行权移交给另一个生成器函数（当前生成器暂停执行），例如：

function* anotherGenerator(i) {
  yield i + 1;
  yield i + 2;
  yield i + 3;
}

function* generator(i){
  yield i;
  yield* anotherGenerator(i);// 移交执行权
  yield i + 10;
}

var gen = generator(10);

console.log(gen.next().value); // 10
console.log(gen.next().value); // 11
console.log(gen.next().value); // 12
console.log(gen.next().value); // 13
console.log(gen.next().value); // 20

当在生成器函数中显式 return 时，会导致生成器立即变为完成状态，即调用 next() 方法返回的对象的 done 为 true。如果 return 后面跟了一个值，那么这个值会作为当前调用 next() 方法返回的 value 值。例如：

function* yieldAndReturn() {
  yield "Y";
  return "R";//显式返回处，可以观察到 done 也立即变为了 true
  yield "unreachable";// 不会被执行了
}

var gen = yieldAndReturn()
console.log(gen.next()); // { value: "Y", done: false }
console.log(gen.next()); // { value: "R", done: true }
console.log(gen.next()); // { value: undefined, done: true }

# 三、for...of

# 1、基本操作

for...of 是 ES6 新增的语法。它可以在可迭代对象（包括 Array，Map，Set，String，TypedArray，arguments 对象等等）上创建一个迭代循环，调用自定义迭代钩子，并为每个不同属性的值执行语句。换句话说，这个语法可以帮助运行迭代器，不必像之前那样每次迭代的时候都调用一些 next() 方法，所以也可以看作是迭代器执行的的语法糖。

下面是使用 for...of 运行迭代器的例子。对于for...of的循环，可以由break、throw 或return终止，可以使用这种方法强行中断迭代器：

function* foo(){ 
  yield 1; 
  yield 2; 
  yield 3; 
}; 

for (let o of foo()) { 
  console.log(o); 
  break; // closes iterator, triggers return
}
//  1

# 2、for...of 与 for...in的区别

简单地说 for...of 进行的是迭代操作，而 for...in 进行的是遍历操作。for...in语句以任意顺序遍历一个对象的可枚举属性。对于每个不同的属性，语句都会被执行。

for...in 循环只遍历可枚举属性。像 Array和 Object使用内置构造函数所创建的对象都会继承自Object.prototype和String.prototype的不可枚举属性，例如 String 的 indexOf() 方法或 Object的toString()方法，这些属性不会被遍历到。

例如，下面的函数接受一个对象作为参数。被调用时迭代传入对象的所有可枚举属性然后返回一个所有属性名和其对应值的字符串。：

var obj = {a:1, b:2, c:3};
    
for (var prop in obj) {
  console.log("obj." + prop + " = " + obj[prop]);
}

// Output:
// "obj.a = 1"
// "obj.b = 2"
// "obj.c = 3"

可以看出，for...in 每次获得的是对象的键名，而 for...of 获得的是对象的键值。for...in 是旧语法，不管是任何对象都可以遍历；for...of 是新语法，有些对象才可以迭代。

注意，数组索引只展示具有整数名称的枚举属性，并且不能保证for ... in将以任何特定的顺序返回索引。f而or ... in循环语句将返回数组所有可枚举属性，包括具有非整数类型的名称的属性和继承的属性。

因为迭代的顺序是依赖于执行环境的，所以数组遍历不一定按次序访问元素。因此当迭代访问顺序很重要的数组时，最好用整数索引去进行for循环（或者使用 Array.prototype.forEach() 或 for...of 循环）。

# 3、为什么有些对象不能迭代

在 JavaScript 中，一些内置类型，如Array或Map具有默认的迭代行为，而其他类型（如Object）没有。到底是什么决定了一个对象能否迭代，当一个对象使用 for...of 进行迭代操作时，到底发生了什么？

实际上，所有可迭代的对象都是遵循可迭代协议的。可迭代协议允许 JavaScript 对象去定义或定制它们的迭代行为, 例如（定义）在一个 for..of 结构中什么值可以被循环（得到）。这个协议具体地说，就是为了变成可迭代对象， 一个对象必须实现 @@iterator 方法, 意思是这个对象（或者它原型链 prototype chain 上的某个对象）必须有一个名字是 Symbol.iterator 的属性。

而这个属性的值是一个对象的无参函数，被返回对象符合迭代器协议。这个属性就是一个生成器，当一个对象需要被迭代的时候（比如开始用于一个for..of循环中），它的 @@iterator方法 将被无参数调用，然后返回一个用于在迭代中获得值的迭代器。

所以说，一个对象进行迭代（比如开始用于一个for..of循环中）的过程，实际上就是在迭代 @@iterator 方法生成的迭代器。那些默认可以迭代的对象都拥有默认的 @@iterator方法。因此，只要也给的对象的 Symbol.iterator 属性设置为一个生成器，那么的对象就可以迭代了。比如：

var myIterable = {}
myIterable[Symbol.iterator] = function* () {
    yield 1;
    yield 2;
    yield 3;
};
[...myIterable] // [1, 2, 3]
