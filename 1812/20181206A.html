# 什么是Sass？

Sass 是一款强化 CSS 的辅助工具，它在 CSS 语法的基础上增加了变量 (variables)、嵌套 (nested rules)、混合 (mixins)、导入 (inline imports) 等高级功能，这些拓展令 CSS 更加强大与优雅。Sass具有以下特色功能：

  • 完全兼容 CSS3
  • 在 CSS 基础上增加变量、嵌套 (nesting)、混合 (mixins) 等功能
  • 通过函数进行颜色值与属性值的运算
  • 提供控制指令 (control directives)等高级功能
  • 自定义输出格式

Sass 有两种语法格式。首先是 SCSS (Sassy CSS) ，这种格式仅在 CSS3 语法的基础上进行拓展，所有 CSS3 语法在 SCSS 中都是通用的，同时加入 Sass 的特色功能。此外，SCSS 也支持大多数 CSS hacks 写法以及浏览器前缀写法 (vendor-specific syntax)，以及早期的 IE 滤镜写法。这种格式以 .scss 作为拓展名。因为这种格式与SCSS更为相近，所以在这里使用SCSS的格式进行编写代码。

另一种也是最早的 Sass 语法格式，被称为缩进格式 (Indented Sass) 通常简称 "Sass"，是一种简化格式。它使用 “缩进” 代替 “花括号” 表示属性属于某个选择器，用 “换行” 代替 “分号” 分隔属性，很多人认为这样做比 SCSS 更容易阅读，书写也更快速。缩进格式也可以使用 Sass 的全部功能。这种格式以 .sass 作为拓展名。

可以将Sass看作是一种类CSS语言，因为它比CSS更加简洁，功能也更加强大，能够给的开发带来便利，所以就要学会使用这门工具。因为浏览器并不原生支持Sass，所以需要使用一些工具对Sass文件进行编译。在这里就是用parcel，parcel可以说是一种能够不安装Webpack就可以使用Webpack的工具。parcel安装完成后，直接在命令行输入：

npx parcel index.html
1

这样就能够自动将index.html中引用的Sass文件翻译为CSS，除此之外parcel还会自动开启一个端口给预览index.html，预览效果会和代码保持同步更新。可以说在学习Sass的过程中，parcel算是一大助力了。

配置好开发环境之后，就步入正题了，来来讲讲一些Sass的一些基础语法，看看它到底比CSS强在哪里。

# Sass基础语法

Sass（SCSS）的基础语法与CSS相比做了许多改进，将很多地方变得更加人性化，使得的代码体验更加舒适。

# 1、选择器嵌套

Sass 允许将一套 CSS 样式嵌套进另一套样式中，内层的样式将它外层的选择器作为父选择器，例如：

#main {
  width: 97%;

  p, div {
    font-size: 2em;
    a { font-weight: bold; }
  }

  pre { font-size: 3em; }
}

# 2、父选择器 &

在嵌套 CSS 规则时，有时也需要直接使用嵌套外层的父选择器，例如，当给某个元素设定 hover 样式时，或者当 body 元素有某个 classname 时，可以用 & 代表嵌套规则外层的父选择器。例如：

a {
  font-weight: bold;
  text-decoration: none;
  &:hover { text-decoration: underline; }
  body.firefox & { font-weight: normal; }
}

# 3、属性嵌套

有些 CSS 属性遵循相同的命名空间 (namespace)，比如 font-family, font-size, font-weight 都以 font 作为属性的命名空间。为了便于管理这样的属性，同时也为了避免了重复输入，Sass 允许将属性嵌套在命名空间中，例如：

.funky {
  font: {
    family: fantasy;
    size: 30em;
    weight: bold;
  }
}

# 4、占位符（Placeholder）选择器

Sass 额外提供了一种特殊类型的选择器：占位符选择器 (placeholder selector)。与常用的 id 与 class 选择器写法相似，只是 # 或 . 替换成了 %。必须通过 @extend 指令调用。这样说得比较不清不楚，举个例子来说明。

常常会将同一个样式运用在不同选择器的元素上，这时候占位符选择器就相当于为这些选择器占据好了位置。在Sass文件被编译为CSS文件之后，就可以发现这些选择器是并列使用这套样式的。比如：

%box {
  box-shadow: 0 0 3px black;
  margin: 10px;
  background: white;
  border-radius: 4px;
}

.nav {
  > ul {
    > li {
      @extend %box;
    }
  }
}
.demo{
  height: 100px;
  @extend %box;
}

# 5、单行注释与多行注释

Sass 支持标准的 CSS 多行注释 /* */，以及CSS不支持的单行注释 //，前者会 被完整输出到编译后的 CSS 文件中，而后者则不会，例如：

/* 这是多行注释
 * 会被完整编译到CSS中 */
body { color: black; }

// 这是单行注释，只在SCSS文件中可见
a { color: green; }
