【归纳】ES6的一些语法糖

广宇 2018-12-19 ES6

今天把自制ui组件的网格系统的内容自己实现了一下，没想到花了那么久，几乎做了一天，效率有点低。在自己尝试的过程中总是能碰到各种各样的问题，在解决问题之后还是可以获得成就感。

总而言之做出来了就是好事，今天晚上顺便把之前看的ES6深入浅出的一些知识整理一下。本篇博客我将会把ES6中函数、对象的一些相关的语法糖分别进行归纳，希望今晚能够写完吧。

# 一、默认参数

平时定义函数有的时候会给参数设置默认值，比如：

function sum(a,b){
  return a+b
}
1
2
3

上面的函数必须传两个参数进去才能进行运算。如果只传一个参数，如 sum(1)，b 的值默认为 undefined，就会的到 NaN 的结果。如果这时设置 b 的默认值为 0，这样即使 sum 只接受一个参数，也能正常运算了：

function sum(a,b){
  b = b || 0
  return a+b
}
1
2
3
4

上面的方法也是在 ES6 之前的惯用的设置参数默认值的方法。虽然也谈不上有多麻烦，但是如果函数需要很多参数，那就显得不方便了。对此 ES6 中新增了这样设置默认参数的方法：

function sum(a = 0, b = 0){
  return a+b
}
1
2
3

这样即便 a、b 的值都是 undefined，在函数中都会使用为 0 的默认值。

需要注意的是，与 Python 中的情况不同，JavaScript 中的函数默认值在函数调用时会被解析，每次都会创建一个新的参数对象。比如：

function push(item,array=[]){
  array.push(item)
  return array
}
push(1)
//[1]
push(2)
//[2]
1
2
3
4
5
6
7
8

两次进行 push 操作的数组不是同一个数组，可以这样理解，函数调用时的 arguments 对象都是不同的，每次函数都会初始化这个空数组。但是在 Python 里面这个空数组只会初始化一次。

下面是一个比较复杂的默认值设定操作，在函数不传参的情况下，参数默认为空对象，而这个空对象中的 size、cords、radius 属性都有自己的默认值：

function x({size = "big", cords = {x:0, y:0}, radius = 25} = {}){……}

# 二、剩余参数

接着考虑刚才的例子，怎样做才能实现不管传多少个参数都可以求和呢？这个问题就麻烦得多了，因为我甚至不知道我要传几个参数。不过在 ES5 中也不是不能实现的，实现方法如下：

function sum(){
  var result = 0
  for(i=0; i<arguments.length; i++){
    result+= arguments[i]
  }
  return result
}
1
2
3
4
5
6
7

就是遍历一下 arguments ，也就是所有参数。虽然也谈不上特别麻烦，还是来看看用 ES6 怎么办。这回给输出结果加个前缀，这样需要求和的参数就要从第二个参数开始了，用上面的方法也就是从 i = 1 开始遍历即可。如果用 ES6 的方法，可以直接借助一个语法糖，获取到函数的所有剩余参数：

function sum(message = "结果为", ...number){
  let result = 0
  result = number.reduce((p,v) => p+v, 0)
  return message + result
}
1
2
3
4
5

其中，...number 代表在 message 之后传的所有参数。number 是一个数组，因此可以使用 reduce 方法，而 arguments 是一个伪数组。借助这种「三个点」的语法，可以将 arguments 转为数组，下面是几种方法：

// ES5 的方法
let arg1 = Array.prototype.slice.call(arguments)
// ES6 的方法
let arg2 = Array.from(arguments)
let arg3 = [...arguments]
1
2
3
4
5

其中，...arguments 即意为 arguments 对象的所有属性。所以这样就相当于把 arguments 对象的所有属性都填入一个空数组中，自然实现了将 arguments 转为数组的效果。

# 三、解构赋值

这一节讲解构赋值。解构赋值也可以叫做解析赋值，也是是一种语法糖。我最初见到的时候已是让我相当困惑，在这里就把几种解构赋值的情况都归纳一下，借助这些例子去了解其中的原理。解构赋值也常常与刚才讲的「三个点」的语法一起使用，这里就一并讲解了。

# 1、对调两个变量的值

a = 1 ;
b = 2 ;
[a,b] = [b,a] ;
1
2
3

这个例子看上去就很好理解。以前对调变量一般都会使用一个中间变量作为过渡，但是现在不需要了。需要注意的是，这里少见地用到了 ";" 号。知道在 JavaScript 里面分号不是必要的，这里用到分号是因为 JavaScript 中的 "[]"、"()" 等括号在句首时会自动上提一行，就可能会出 bug。

# 2、「三个点」语法

# 1）展开操作

比如：

let array1 = [1,2,3,4,5,6]
let [a,b,c,...array2] = array1
a   // 1
b   // 2
c   // 3
array2  // [4,5,6]
1
2
3
4
5
6

这是一种会让习惯了ES5的风格的人感到匪夷所思的方法，这样同时给 a、b、c、array2 同时赋值了，其中 array2 使用了「三个点」的语法，代表数组中剩下的所有值。如果不需要 a、b、c 这三个值，可以这样写：

let [,,,...array2] = array1
1

这样的写法看起来很随意，不过还是很管用的，作用就是将 array1 中第三个值之后的所有值放在一个新数组里。除了利用这个语法提取值，还可以利用这个语法给数组前后添加值，比如：

let array3 = [0,...array1,7]
//  [0,1,2,3,4,5,6,7]
1
2

这样就相当于把 array1 的内容展开来再放到 array3 里，所以叫做「展开操作」，以往给数组前后添加值一般都是 concat 方法，比如 [0].concat(array1).concat([7]) ，新方法显得快得多了。

# 2）对象的浅拷贝

借助这种方法，还能实现对象的浅拷贝。来回忆一下什么是浅拷贝，下面是一个例子：

let objA = {name: 'a',message:{age:18}}
let objB = Object.assign({},objA)
objA.name='b'
// "b"
objB.name
// "a"
objA.message.age = 20
objB.message
// {age:20}
1
2
3
4
5
6
7
8
9

一旦 objB 的内容改变，objA 的内容并不随之改变，这样的情况就可以称作是浅拷贝。但是使用上述的方法实现浅拷贝并不稳妥，因为如果这样拷贝对象时只会拷贝对象在内存中存放的地址，所以就实现不了浅拷贝的效果。对此，在 ES6 中可以这样实现浅拷贝：

let objA = {message:{age:18}}
let objB = {...objA}
objA.message.age = 20
objB.message
// {age:18}
1
2
3
4
5

可以这样理解，利用「三个点」语法就是相当于将这个对象所有属性的值取出来了。上面的例子就是将 objA 中的属性放到一个空对象里，再赋给 objB，从而实现了浅拷贝。

# 4）对象的合并

同样地，利用这个方法，还能够实现对象的合并。以往的方法仍然是使用 Object.assign 方法，新方法就显得方便得多：

let objA = {p1:1, p2:2}
let objB = {p1:111, p2:3}
//  老方法
let objC = Object.assign({},objA,objC)
//  新方法
let objD = {...objA,...objB}

# 3、提取变量

经常需要将对象中的值提取到变量里面以便操作，在 ES6 中同样也给提供了贴心的语法糖，这和刚才提到的「展开操作」原理有些相似，只是数组被换作了对象而已：

var frank ={
  name: "frank",
  age: 18,
  gender: "male"
}
var {name, age, gender} = frank
//  等价于
var name = frank.name
var age = frank.age
var gender = frank.gender

上面还给出了 ES6 语法和传统方法的对比，可以说解构赋值是非常方便了。之前的「展开操作」中的赋值是与值在数组中的位置挂钩的，这里因为是对象，所以就要通过 key 来提取数组中的内容。上面就是在 frank 对象中提取 name、age、gender 等 key 的值，如果这个对象中没有这个 key，也是提取不了的。

这样的解构赋值也可以结合默认值一起使用，比如：

var [a = 5,b = 7] = [1]
a   // 1
b   // 7
1
2
3

上面的例子中，5 和 7 就分别是 a 与 b 的默认值，同时 a 被赋值为 1 ，b 因为没有没有值可以赋给它，所以默认为 1。如果想取到数组中的某个值，也可以这样操作，这是一个例子：

var fn = function(){
  return [1,2,3]
}
var [a,,b] = f()
a   // 1
b   // 3
1
2
3
4
5
6

这样 a、b 之间的位置的值就被跳过赋值了。

之前提取对象中的值的变量，变量的名称就是这个值在对象中 key 的值。如果不想变量叫这个名称怎么办？其实在提取变量的同时就可以对变量进行命名了：

var {name:"姓名", age:"年龄", gender:"性别"} = frank
姓名  // frank
年龄  // 18
性别  // male
1
2
3
4

还可以再进一步给这些变量设置默认值，这样就可以避免对象中没有这些 key 造成提取处 undefined 的尴尬情况：

let {name:"姓名" = 'Tom'} = frank
1

如果 Frank 中无 name 属性，姓名的值就默认为 Tom。

# 四、对象属性的加强

在一些情况下，对象属性的语法可以得到一些简化，比如：

var x = 1
var y = 2
var obj ={
  x,
  y
}

函数属性也得到了一个简单的简化效果：

var obj2 = {
  say: function(){}
}
// 等价于
var obj2 = {
  say(){}
}
