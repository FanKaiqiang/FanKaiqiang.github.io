【归纳】ES6 中的模块

广宇 2018-12-28 ES6

昨天我刚刚把自制 UI 毕设的课程看完，这几天着手把之前想写而没写的博客写一下。希望这几天能够高产一些吧，不然这个月就完不成既定目标了。

本篇博客的主题是 ES6 中的模块，希望能够通过这次博客巩固一下我新学的这些知识，同时便于日后查阅。

# 一、引言

在 ES5 中，JavaScript 是没有模块的说法的。但是在同一时期的其他语言，比如 Python、PHP、Java 等语言，都有模块的概念。那么什么是模块呢，为什么需要模块？

在页面中使用 JavaScript 时，都是需要在页面中使用 <script> 标签来引入 JS 的内容。有的时候可能会引入多个 JS 文件，而不是把这多个文件整合成一个文件使用，为的就是让这些文件中的代码显得逻辑清晰，同时互不干扰。可以这样理解，将做一件事的 JS 代码放在一个模块（文件）里，另一件事的代码就放在另一个模块（文件）里，这样就实现了初步的模块化。

各个模块是共享同一个全局变量的，这样就可能产生冲突。为了使这些模块的内容互不干扰，在 ES6 之前，会使用立即执行函数将整个模块包裹起来，这样在每个模块中声明的变量就是局部变量，其他模块访问不了。如果需要将模块的一些内容导出到模块外部，就需要借助全局变量 window。将自己需要导出的内容暴露在全局变量中，其他模块就可以将这些内容进行导入了。这样就实现了模块间的联系。

现在有了 ES6 新语法，已经不再需要立即执行函数，二十使用新版的块级作用域。而在 ES6 中，JavaScript 正式引入了模块的概念，模块间的交流也不必再借助 window 了。接下来就来看看，ES6 中给带来的有关模块的一些语法。

# 二、导出（export）

export 语句用于在创建 JavaScript 模块时，从模块中导出函数、对象或原始值，以便其他程序可以通过 import 语句使用它们。export 有几种不同的导出方式，这里分别列出。

# 1、使用命名导出

在模块中，可以使用以下代码：

// module "my-module.js"
function cube(x) {
  return x * x * x;
}
const foo = Math.PI + Math.SQRT2;
export { cube,foo };

import { cube, foo } from 'my-module.js';
console.log(cube(3)); // 27
console.log(foo);    // 4.555806215962888
1
2
3

在使用命名导出时，可以在导出的变量进行重命名，这样在其他模块引入这些变量的时候就需要使用新的名字：

export { cube as name1,foo as name2}
1

上面的例子还可以这样改写，这样变量在声明的同时也被命名导出了：

// module "my-module.js"
export function cube(x) {
  return x * x * x;
}
export const foo = Math.PI + Math.SQRT2;

# 2、使用默认导出

如果要导出一个值或模块中的返回值，就可以使用默认导出：

// module "my-module.js"
export default function cube(x) {
  return x * x * x;
}
1
2
3
4

在另一个脚本中，可以直接导入默认导出。值得注意的是，可以使用任何名称导入默认导出：

// module "my-module.js"
import FanKaiqiang from 'my-module';
console.log(FanKaiqiang(3)); // 27​​​​​
1
2
3

可以使用 function 声明函数（可以是匿名函数）并默认导出，但是不能使用 var，let 或 const 声明变量时同时进行默认导出export default。如果需要对这些变量默认导出，可以这样做：

// module "my-module.js"
let cube = function (x) {
  return x * x * x;
}
const foo = Math.PI + Math.SQRT2;
export default cube;
export {foo};
1

上面的代码默认导出 cube，命名导出 foo。两句导出的代码可以合并为：

export {cube as default,foo}
1

这样 cube 就被默认导出了。

# 3、模块重定向

可以使用 "*" 在模块中再次导出其他模块中已经导出的内容，比如：

export * from './other-module';
1

这个语法不会导出 other-module 模块中的默认导出，如果要导出此模块的默认导出，可以这样做：

import mod from "./other-module";
export default mod;
1
2

或者：

export {default} from './other-module';
1

# 三、导入（import）

import 语句用于导入由另一个模块导出的绑定。无论是否声明了 strict mode，导入的模块都运行在严格模式下。下面是几种不同的导入方式：

# 1、导入命名导出

# (1) 导出所有命名导出

使用 "*" 可以导入模块中的所有具名导出，比如：

import * as myModule from '/modules/my-module.js';
1

导入后 myModule 变量加入当前作用域，其中包含来自位于 /modules/my-module.js 文件中导出的所有命名导出。比如在之前命名导出了一个变量 cube，在这里可以这样使用它：

myModule.cube(6)
1

# (2) 导出具体的命名导出

给定一个名为 myExport 的对象或值，它已经从模块 my-module 命名导出。下面的导入将会把 myExport 变量插入当前模块作用域：

import {myExport} from '/modules/my-module.js';
1

导入多个命名导出：

import {foo, bar} from '/modules/my-module.js';
1

导入时可以先重命名再导出。例如，将 newName1、newName2 插入当前作用域。

import {foo as newName1,bar as newName2} from '/modules/my-module.js';
1

# 2、单纯地导入模块

使用 import 导入模块时，相应地会执行模块中的代码。如果只需要执行这些代码，并不需要导入模块中的变量，就可以这样做：

import '/modules/my-module.js';
1

# 3、导入默认导出

在模块具有默认导出时，可以使用 import 语句来导入此模块的默认值。最简单的用法是直接导入默认值：

import myDefault from "my-module";
1

也可以同时将 default 语法与上述用法（命名空间导入或命名导入）一起使用。在这种情况下，default 导入必须首先声明。 例如：

import myDefault, * as myModule from "my-module";
1

或者

import myDefault, {foo, bar} from "my-module";
1

# 四、注意事项

如果要在浏览器中直接使用模块语法，在 <script> 标签需要添加一条属性：

<script type="module" src="./1.js"></script>
1

模块语法作为 ES6 的新增语法，在一些浏览器中还不支持，但是它在许多转换器中已经实现，例如 Traceur Compiler ， Babel ， Rollup 或 Webpack。所以需要借助上述工具对使用了这些新语法的代码翻译成旧语法，以保证代码的兼容性。
