【归纳】箭头函数

广宇 2018-12-5 JavaScript ES6

这个月仍然是计划写20篇博客，每三天写两篇博客。不过因为系统课结束，可能会找不到那么多主题来写。所以今天在大致地过完一遍Vue的基础文档之后，看完了一节ES6深入浅出的一节课——关于箭头函数的内容。

这也是今天博客大致要讲的主题，箭头函数也是早就接触过的知识了，所以这篇博客内容会较少，完全是整理自我的笔记，算是水了一篇博客吧。

# 一、函数声明的方式

早在ES3，就有使用function关键字声明函数的方法了。之前在刚刚学习函数的时候就写过关于函数的博客:【归纳】JS函数，里面就列出了所有函数声明的方式。其中可以使用function关键字声明匿名函数与具名函数：

function x(){}  //  具名函数

let y = function(){}  //  匿名函数

除此之外，在之前的博客里早已提到过箭头函数，再来重新回顾一遍：

let f1 = (x1,x2)=>{
  return x1 + x2
}

let f2 = (x1,x2)=> x1 + x2

let f3 = x3 => {console.log(x3)}

箭头函数是ES6推出的声明函数的新语法，这种语法起源于CoffeeScript中的「胖箭头函数」，看起来就比ES3的老方法要简洁得多。去掉了function关键字的声明，然后使用形如"=>"的箭头来连接参数()与函数体{}。除此之外，在函数体只有一句return的情况下，可以将包裹函数体的大括号和return同时省略。如果传入的参数只有一个，那么包裹参数的括号也可以省略。箭头函数也是匿名函数。

由此看来，ES6的箭头函数语法就是在想尽办法，把ES3的函数声明方法简化再简化。但是总感觉仅仅为了省略这几个字，就创作出这个新语法，这虽然是大家喜闻乐见的，但是能给带来的帮助也是有限的。因此，箭头函数能给带来的方便也远不止于此，箭头函数还有肯定什么其他更有内涵的优势。

# 二、箭头函数的 this

直接从this这个角度来入手这个问题。说到函数就免不了要谈this，同样是早在开始学习函数的时候我就开始接触到this了，前段时间我也专门为this的相关知识点做了一篇博客归纳：【归纳】this指的是什么。this是初学者比较迷糊的一个问题，一个从天而降的this总是可以把搞得晕头转向。举几个简单的例子：

obj = {
  hi: function(){
    console.log(this)
  }
}

obj.hi()

上面就有个this，这个this就是从天而降的，代码里也没说它是从哪来的。之前说过，this就是使用call()方法调用函数的第一个参数，但是上面obj.hi()没有使用call()方法调用。那么this既然作为一个参数，是怎么渗透进参数里面的呢？

知道，在传参数给函数之后，所有参数的值都会被推入arguments数组中。比如第一个数组为arguments[0]就是第一个参数，一次类推。而this是早在arguments之前就传给这个函数的参数，这一点在使用call()方法的时候可以看出来。它的值是由调用这个函数时函数前面的"."决定的，以上面的内容为例，obj.hi()里的this就是obj，这个obj就可以假设它为"arguments[-1]"。

当然，「第-1个参数」是虚构的一个概念，在每个函数调用后，在没指定this的情况下，其实都会把这个"."前面的内容作为「第-1个参数」传给函数。即便是function x(){}这样的空函数也一样，在以x()的方式调用函数之后，因为x前面没有"."，所以window被作为「第-1个参数」传进函数了，所以这个函数里的this等于window。

再来看看一个有this带来的问题，下面是一个例子：

    $(this.el).on('click',function(){
      this.onClick()
    })
  },
  onClick: function(){
    console.log('这里 开心')
  }
}

上面有很多this，根据之前的推断，很容易就可以知道init方法里面的this就是controller。init建立了对#app的点击事件的监听，一旦触发点击事件，就调用function(){ this.onClick() }函数。那么问题来了，看不到这个函数的调用，也就看不到这个函数"."前面的内容，也就不知道这个函数里面的this指的是什么。从代码内容来看，是希望此时this指的是controller，因为想要调用onClick方法。

但是事实上这里的this指的是当前点击的元素。当然这也不是凭空冒出来的，on方法的具体内容在jQuery的源码里面，在哪里就可以看到this.onClick函数是怎么调用的了。但是这根本不是想要的，所以用到了这样的办法：

var controller = {
  el: '#app',
  init: function(){
    var that = this
    $(this.el).on('click',function(){
      that.onClick()
    })
  },
  onClick: function(){
    console.log('这里 开心')
  }
}

这样就可以清楚的看见onClick方法的执行了，而且此时的that就等于init里面的this。使用这种手段，就可以稳住this了。但是，如果使用箭头函数，就能完美解决这个问题：

var controller = {
  el: '#app',
  init: function(){
    $(this.el).on('click',()=>{
      this.onClick()
    })
  },
  onClick: function(){
    console.log('这里 开心')
  }
}

这是因为箭头函数具有并不会接受「第-1个参数」的特性，也就是不接受this，所以箭头函数内部的this的值就会「继承」其外部作用域的this，就和函数中调用一个在函数内部没有定义的变量一样。除此之外，即便是强行使用call方法给箭头函数指定this也是徒劳的，比如：

this === window //true
function f1(){
  console.log(this)
}
f1.call({name.frank}) //{name.frank}
let f2 = ()=>{
  console.log(this)
}
f2.call({name.frank}) //window

所以，箭头函数就是一个纯粹的函数，弱化了this的用法。箭头函数的使用也使得JavaScript的表现力得到了增强。因此，在用不着本函数传入的this时，就尽量使用箭头函数。当然，如果需要用到的内容是依赖于this的，就不能使用箭头函数了。比如我今天过了一遍Vue的文档就提到了这一点：

不要再实例属性或者回调函数中（如vm.$watch('a',newVal=>this.myMethod())）使用箭头函数。因为箭头函数绑定父上下文，所以this不会像预想的一样是Vue实例，而是this.myMethod未被定义。

# 三、总结

补充于 2019年3月7日 09:43:30

箭头函数内部没有constructor方法，也没有prototype，所以不支持new操作。

箭头函数的 this 始终指向函数定义时的 this，而非执行时。

箭头函数不绑定arguments，取而代之用rest参数...解决：

let C = (...c) => {
  console.log(c);
}
C(3,82,32,11323);  // [3, 82, 32, 11323]
