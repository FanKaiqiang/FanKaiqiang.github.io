【归纳】Sass与CSS的变量与运算

广宇 2018-12-11 Sass CSS

这两天学校停电，昨天博客也没写，今天早上也因为停电耽误了一些时间。还好下午已经来电，不然只能和大家打一下午牌了。今天的话就继续把Sass中变量以及运算的部分归纳一下，顺便横向比较一下CSS中变量与运算的用法。

因为这两方面的知识我目前都谈不上很熟悉，所以这次在编写博客的过程中，我也会查找一些文档和资料，边学边写。

在 CSS 属性的基础上 Sass 提供了一些名为 SassScript 的新功能。SassScript 可作用于任何属性，允许属性使用变量、算数运算等额外功能。这些操作虽然被称作SassScript，但是并不是一门独立于Sass的一门新的语言，可能因为和JavaScript中的操作变量与算数运算有些相似，所以才被取了这个名字。

至于CSS，在2017年3月微软也宣布 Edge 浏览器将支持 CSS 变量，因此目前所有主要浏览器已经都支持原生CSS的变量与运算了。本篇博客主要参考了Sass中文文档与阮一峰的CSS 变量教程，这篇文章就综合地把原生CSS与Sass的区别在这两方面比较一下。

# 变量

SassScript 最普遍的用法就是变量，变量以美元符号开头，赋值方法与 CSS 属性的写法一样：

$width: 5em;
1

直接使用即调用变量：

#main {
  width: $width;
}
1
2
3

变量支持块级作用域，嵌套规则内定义的变量只能在嵌套规则内使用（局部变量），不在嵌套规则内定义的变量则可在任何地方使用（全局`变量）。将局部变量转换为全局变量可以添加 !global 声明：

#main {
  $width: 5em !global;
  width: $width;
}

#sidebar {
  width: $width;
}

而原生CSS的变量声明则与Sass有所不同，声明变量的时候，变量名前面要加两根连词线（--）。它们与color、font-size等正式属性没有什么不同，只是没有默认含义。所以 CSS 变量（CSS variable）又叫做"CSS 自定义属性"（CSS custom properties）。因为原生CSS的变量功能出得比较晚，$与@已经分别被Sass和Less用掉了。为了不产生冲突，所以原生CSS用两根连词线（--）表示变量:

:root {
  --foo: #7F583F;
  --bar: #F7EFD2;
}

而在读取变量的时候，则需要使用到var()函数。var()函数还可以使用第二个参数，表示变量的默认值。如果该变量不存在，就会使用这个默认值：

a {  
  --primary-color: red;
  /* var()函数还可以用在变量的声明 */
  --logo-text: var(--primary-color);
  color: var(--primary-color, #7F583F);
  text-decoration-color: var(--logo-text);
}

原生CSS变量的作用域与Sass有些区别。Sass中的变量有点像JavaScript，支持块级作用域。而CSS变量更像是元素的一个属性，变量的作用域就是它所在的选择器的有效范围。读取的时候，优先级最高的声明生效。这与 CSS 的"层叠"（cascade）规则是一致的。下面是一个例子，其中:root选择器意为根元素：

<style>
  :root { --color: blue; }
  div { --color: green; }
  #alert { --color: red; }
  * { color: var(--color); }
</style>

<p>蓝色</p>
<div>绿色</div>
<div id="alert">红色</div>

.foo {
  --side: margin-top;
  /* 无效 */
  var(--side): 20px;

# 运算

SassScript 支持 6 种主要的数据类型，主要包括数字、字符串、颜色、布尔值、空值（null）、数组、map。所有数据类型均支持相等运算 == 或 !=，此外，每种数据类型也有其各自支持的运算方式。

SassScript 支持数字的加减乘除、取整等运算 (+, -, *, /, %)，如果必要会在不同单位间转换值。此外，圆括号可以用来影响运算的顺序：

p {
  width: (1in + 8pt) * 3;
}
1
2
3

/ 在 CSS 中通常起到分隔数字的用途，SassScript 作为 CSS 语言的拓展当然也支持这个功能，同时也赋予了 / 除法运算的功能。也就是说，如果 / 在 SassScript 中把两个数字分隔，编译后的 CSS 文件中也是同样的作用。以下三种情况 / 将被视为除法运算符号：

  • 如果值，或值的一部分，是变量或者函数的返回值
  • 如果值被圆括号包裹
  • 如果值是算数表达式的一部分

因此，需要根据自己的需求正确使用 / 号。下面是一些例子：

p {
  font: 10px/8px;             // 分隔数字，不是除法
  $width: 1000px;
  width: $width/2;            // 值是变量
  width: round(1.5)/2;        // 值是函数返回值
  height: (500px/2);          // 值被圆括号包裹
  margin-left: 5px + 8px/2px; // 值是算数表达式的一部分
}

如果需要使用变量，同时又要确保 / 不做除法运算而是完整地编译到 CSS 文件中，只需要用 #{} 插值语句将变量包裹。使用 #{} 可以避免 Sass 运行运算表达式，直接编译 CSS。

p {
  $font-size: 12px;
  $line-height: 30px;
  font: #{$font-size}/#{$line-height};
}

而在原生CSS中，要想进行数值运算，就需要使用到calc()函数了。这个函数就接受一个运算表达式作为参数，表达式中的操作数可以使用任意语法种类的长度。如果你愿意，你可以在一个表达式中混用多种不同的长度单位。在需要时，你还可以使用小括号来调整计算顺序。下面是一些例子：

.foo {
  --widthA: calc(100% - 80px);
  --widthB: calc(var(--widthA) / 2);
  --widthC: calc(var(--widthB) / 2);
  width: var(--widthC);
}

需要注意的是，+ 和 - 运算符的两边必须始终要有空白符。比如 calc(50% -8px) 会被解析成为一个无效的表达式：一个百分比后跟一个负数长度值。而 calc(8px + -50%) 会被解析成为一个长度后跟一个加号再跟一个负百分比。* 和 / 运算符不需要空白符，但考虑到统一性，仍然推荐加上空白符。

在原生CSS中，如果变量值是数值，不能与数值单位直接连用。下面代码中，数值与单位直接写在一起，这是无效的。必须使用calc()函数，将它们连接：

.foo {
  --gap: 20;
  /* 无效 */
  /* margin-top: var(--gap)px; */
  margin-top: calc(var(--gap) * 1px);
}

在Sass中，+ 还可以用于连接字符串。Sass中的字符串是否有引号是不同的，如果有引号字符串（位于 + 左侧）连接无引号字符串，运算结果是有引号的，相反，无引号字符串（位于 + 左侧）连接有引号字符串，运算结果则没有引号。在有引号的文本字符串中使用 #{} 插值语句可以添加动态的值：

p {
  cursor: e + -resize;
}

p:before {
  content: "Foo " + Bar;
  font-family: sans- + "serif";
}

p:after {
  content: "I ate #{5 + 10} pies!";
}

因为CSS变量并不能用作属性名，所以在CSS中的字符串都是带引号的字符串。下面是原生CSS中字符串变量与字符串拼接的例子，并不需要用到+号，直接讲变量连字符串即可：

body:after {
  --bar: 'hello';
  --foo: var(--bar)' world';
}

正是因为CSS中字符串都是带引号的，如果变量值并非字符串类型的属性，就不能写成字符串。比如下面的无效的代码中'20px'是被当作字符串看待的，自然发挥不出数值的作用：

/* 无效 */
.foo {
  --foo: '20px';
  font-size: var(--foo);
}

/* 有效 */
.foo {
  --foo: 20px;
  font-size: var(--foo);
}

比较值得一提的是，Sass还支持颜色值运算。颜色值的运算是分段计算进行的，也就是分别计算红色，绿色，以及蓝色的值.如果颜色值包含 alpha channel（rgba 或 hsla 两种颜色值），必须拥有相等的 alpha 值才能进行运算，因为算术运算不会作用于 alpha 值:

p {
  $color1: #010203 + #040506;
  $color2: #010203 * 2;
  $color3: rgba(255, 0, 0, 0.75) + rgba(0, 255, 0, 0.75);
}
// 运算结果
p {
  $color1: #050709;
  $color2: #020406;
  $color3: rgba(255, 255, 0, 0.75);
}

正是基于这些颜色操作的功能，Sass还提供了许多操作颜色的函数，如 opacify 、 transparentize 等。就不再这里过多讲解了。

以上就是本篇博客关于Sass与CSS的变量与运算的所有内容。CSS与Sass的变量与运算的方式各有春秋，总的来说，Sass在这方面表现得更灵活，毕竟也是称作SassScript的。但是原生CSS既然支持这些功能，让不用再借助什么其他工具，也给带来了极大地方便。因此对两方面都有所了解，才能使在日后的开发中游刃有余。
