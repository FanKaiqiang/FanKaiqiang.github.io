【随笔】hexo博客真的好坑

广宇 2018-9-29 Hexo

今天在外面出差，一整天没用空学习。晚上待在酒店里正好也有些空闲，就把昨天遇到的关于hexo博客的问题解决了。所以这次干脆就把我刚刚解决的问题写成博客，记录下来，以防下次遇到束手无策。

试一下音乐播放器有没有效果，顺便调节一下博客里的气氛：

继续回到刚才的问题，我昨天晚上写完那篇博客的时候，hexo d发现总是会出现一个错误：

然后我就上网查资料。结果上网一看，好嘛，网上他们用hexo博客遇到这种问题的我还没有找到。但是类似的解决办法我倒是试了一下：

没有用。我又自己瞎几把试其他的方法，都没有用。我决定在微信群里求助:

老师很干脆地劝退我，这也是我的意料之中的事，因为他本就不推荐使用hexo的。我其实也觉得hexo超级麻烦，但是我感觉我已经在这上面花了一些心血，一旦弃之，心里有点不是很甘心。

我于是决定拆掉重来，我新建了一个文件夹，重新导入我在GitHub上备份的博客存档：

导入回来试了一下hexo d，结果诶，还真的可以。算是成功了一半，至少昨天的成果可以发布了。但是其中有一个问题，就是这个主题他是没有备份在GitHub上的，所以刚刚把博客clone下来的时候是打不开的，因为没有配置主题。所以主题要自己在本地留一个底。

然后还有一个问题，这个问题是关于图片显示的。我发现我的博客里的图片在首页能正常显示，点进去就不行了。后来用开发者工具一看，才发现是路径问题。这时我想起来，是个插件没有安装：

npm install hexo-asset-image --save
1

后面过一会就好了，想是没有装这个插件的原因了。

至此我的博客又可以正常运转了。总的来说用我这个方法并不是特别慢，虽然看起来特别愚笨，应该有大佬是知道正常的方法的，这里就献丑了。不过尽管问题解决了，但是我对hexo的怨念一时半会是消散不了了。
