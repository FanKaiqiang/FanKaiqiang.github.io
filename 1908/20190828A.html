【归纳】柯里化与高阶函数

广宇 2019-8-29 JavaScript

本来这篇博客的内容和前两篇博客都是属于一节课的内容，我这儿竟然分了三篇博客来写，可见这节课的内容真的有点多😏。今天在上班时间来开这篇博客，也并不是想上班时间就把这篇博客写完，争取周六前写完吧，周六还想写些其他的东西。

本篇博客来讲函数的一些其他内容，柯里化和高阶函数，两个看起来「高档」的词语。这篇博客写完，JS 函数的复习也告一段落了，成果还是比较满意的，毕竟肝了三篇博客。

# 一、柯里化

先不说废话，遇到不明白的地方先百度百科：

在计算机科学中，柯里化（Currying）是把接受多个参数的函数变换成接受一个单一参数(最初函数的第一个参数)的函数，并且返回接受余下的参数且返回结果的新函数的技术。

柯里化就是让接受多个参数的函数变换成接受一个单一参数的函数，但也不止于此。这里有两个例子，其中，第二种方法才是本文要讲的柯里化。柯里化一个函数需要这个函数返回结果也是函数，以便接受剩下的参数，并返回相应的值：

const add = (a, b) => a + b;

const noCurrying = ({a, b}) => a + b;
const currying = a => b => a + b;

上面的柯里化看起来挺简单，不妨把难度提升一下，也可以借此提出面试题。假设addTwo接受两个参数，addThree接受三个参数，addFour接受四个参数，需要写出一个currify函数，使得它们分别接受 2、3、4 次参数，即实现如下的效果：

const addTwo = (a, b) => a + b;
const addThree = (a, b, c) => a + b + c;
const addFour = (a, b, c, d) => a + b + c + d;
currify(addTwo)(1)(2); // 3
currify(addThree)(1)(2)(3); // 6
currify(addFour)(1)(2)(3)(4); // 10

这里的难处是，不知道函数要接受几个函数才能结束，就不知道函数应该返回最终结果，还是一个需要再次接受参数的函数。当时这一点也是难住我了，可见我的基础知识并不牢固。实际上，是有方法知道柯里化的函数应该接受的参数的数量的，那就需要借助函数的length属性。之前是没有了解过的，函数也有长度？来看 MDN 怎么说：

length 是函数对象的一个属性值，指该函数有多少个必须要传入的参数，即形参的个数。形参的数量不包括剩余参数个数，仅包括第一个具有默认值之前的参数个数。与之对比的是，arguments.length是函数被调用时实际传参的个数。

前几天的博客里面讲了arguments.length是实参的个数，不过看来形参个数却没有那么简单，上面也有几句话说得不清不楚。看一下下面的例子，就知道原来使用剩余参数预发传入的参数不被计入，设置了默认值的参数和在其之后传入的参数也不会计入。翻译 MDN 的人语文似乎不怎么好，一开始我都没看懂😒：

console.log(Function.length); /* 1，Function本身也是函数 */

console.log((function() {}).length); /* 0 */
console.log((function(a) {}).length); /* 1 */
console.log((function(a, b) {}).length); /* 2 一次类推 */

console.log((function(...args) {}).length); 
// 0，因为使用剩余参数预发传入的参数不被计入

console.log((function(a, b = 1, c) {}).length);
// 1，只有一个参数在设置了默认值的参数之前传入
// c 在其之后，因此不被计入

看来Function.length这个属性的口味还比较挑，看来具有默认值的参数，还是写在后面比较好。以前觉得传参数的顺序这种细节无所谓，现在看来关系还是不小的。

言归正传，这里就不多说了，直接上例子：

const currify = (fn, params = []) => {
    return (arg) => {
        const myParams = params.concat(arg);
        if (myParams.length === fn.length) {
            return fn(...myParams);
        } else {
            return currify(fn, myParams);
        }
    };
}

为了避免改变传入的参数，这里使用的是concat而不是push，因为concat会生成新数组，而push是在原数组上操作。通过函数currify，将函数进行封装，原本需要一次性传入的参数，改为一个一个传入，再依次接入数组中。当数组长度等于函数「长度」时，再以数组内容为参数对原函数进行调用。需要注意的是，如果是函数「长度」不确定的函数（比如可以接受任意个数参数的函数），使用这个函数是没法柯里化的。

# 二、高阶函数

上面的currify实际上就是本篇博客要讲的高阶函数，来看看高阶函数的定义：

高阶函数

在数学和计算机科学中，高阶函数是至少满足下列一个条件的函数：1、接受一个或多个函数作为输入；2、输出一个函数

简单来说，把函数作为参数或返回值的函数，就是高阶函数。currify两个条件都满足。高阶函数其实不是一个复杂的概念，这里只能简单地讲一讲。

currify可以做一些升级。对于上面的currify，如果需要封装的函数参数个数确定，但并不想一次一次传入，而是每次传入的数量不定，那应该怎样改写原来的代码呢？不过这样就不是柯里化了，因为每次传入的参数并不是一个了：

const noCurrify = (fn, params = []) => {
    return (...args) => {
        const myParams = params.concat(...args);
        if (myParams.length === fn.length) {
            return fn(...myParams);
        } else {
            return noCurrify(fn, myParams);
        }
    };
}
noCurrify(addTwo)(1)(2)
// 3
noCurrify(addTwo)(1, 2)
// 3

noCurrify在处理参数上更加自由了。对于这个函数，在写法上也可以进一步优化。下面的函数实际上也形成了尾递归，每次函数返回值都是一次递归操作：

const noCurrify = (fn, params = []) =>
    (...args) =>
        params.length + args.length === fn.length
            ? fn(...myParams, ...args)
            : noCurrify(fn, [...myParams, ...args]);

在 JS 中，高阶函数其实是随处可见的。比如，call与bind，在他们接受的参数为函数时，也是就成了高阶函数。这一点体现得并不明显。Array.prototype.sort()就是很明显的接受函数作为参数的函数，sort接受的函数用于指定排序的顺序，因此sort也是高阶函数。此外，诸如map、forEach、reduce、filter等一系列数组操作方法都属于高阶函数。

高阶函数的思维运用非常广泛，在工作中也很常见。与此相似地，React 中有着高阶组件的概念。React 文档中是这样介绍的：

高阶组件（HOC）是 React 中用于复用组件逻辑的一种高级技巧。HOC 自身不是 React API 的一部分，它是一种基于 React 的组合特性而形成的设计模式。具体而言，高阶组件是参数为组件，返回值为新组件的函数。

组件是将 props 转换为 UI，而高阶组件是将组件转换为另一个组件。不过需要注意的是，高阶组件接受与返回的组件都是类组件，所以并不符合高阶函数的要求，接受与返回函数组件的函数又不能成为高阶函数了。
