【记录】移动端滑动穿透问题解决

广宇 2019-12-13 JavaScript CSS

真的有好些日子没有写博客了，摸了一两个月鱼，没想到要在双十二的晚上加班来写这篇博客。这也是因为正好今天可能是难得的清闲，也正好之前在组会上我向大家提到了这个问题，虽然问题比较捞，但是既然提出来了就要解决，所以就干脆把解决这个问题写的 wiki 搬运过来，再修改修改，就当作是这次的博客了。

这个问题早在两个月前，测试就提 jira 给我了，因为问题不是那么浅显地就能解决，再加上自己还比较垃圾，就一直拖着没搞。直到最近一次组会，组长要求大家分享问题，我才把这个问题提出来。今天花了大半天，倒是总算搞清楚了这其中的缘由。这里就把整个过程记录一下吧。

# 问题的由来

这个问题是发生在商详页面的各种弹窗上的，测试发现只要弹窗是可滚动的，当弹窗内容滚动到顶部或底部的话，如果再做继续滚动的手势，这个时候弹框的背景（也就是 body）就跟着滚动了，体验非常不好，还好只是测试发现了。遇到这个问题的时候我是有点懵逼的，这个问题完全没有了解过啊。最初的思路是阻止弹窗的触摸事件冒泡到 body 上，但是根本没有卵用。而且当时用我自己的手机，在本地调试也看不出问题来，但是在线上又有这个问题，就很迷惑。所以也就一直没去解决这个问题，这个问题也就在线上一直存在。大概是这样的一个场景：

后面呢为了开组会有问题可以说，就把这个我一直拖着没解决的问题提出来了。大家一看，小问题。所以我再会上也没得到什么启发，但是小问题我也得解决嘛，而且下次会上肯定要给个结论。后面测试也催的急了，毕竟拖了要两个月了，就开始试着去解决。这个时候在网上看到一个包，叫做tua-body-scroll-lock，顺手就试试看，结果一试还真可以。叫测试测测也没问题。不过后来有发现现在商详的 sku 弹窗没有穿透的问题，就想干脆先不引入外部包，用现成的解决方法，先凑合着上吧，所以还是没有用新包。

后面组长问我，这个问题看你上线了啊，怎么解决的呢？我一时也说不出所以然来，毕竟不管是现成的 sku 弹窗用的方法，还是外部包里用到的办法，代码我都没看，不知道是怎么搞好的，反正就搞好了。这样肯定是不行的，到后面交不了差的。后面我大概整理了一下为什么咱们之前用的方法是有问题的原因，因为涉及到一些代码，这篇博客里就不仔细分析为什么会出问题了，直接来讲讲原因吧。

现在用的方法是应该是在项目基础库里的，通过阻止 body 的 touchmove 的默认行为，因为 touchmove 的默认行为会触发浏览器的滚动，只要禁掉 touchmove 页面就不会滚动了。但是这样得话弹窗界面的滚动也会被影响，所以这里需要额外监听弹窗元素的 touchmove 事件，允许弹窗内容的 touchmove 的默认行为。但是如果设置了弹窗内部允许滚动的话，那么在弹窗内部滚动到尽头时，如果继续做滚动的手势，就会触发到body的滚动，也正是前文提到的现象。这里借用网上的一张图：

至于如何开关 touchmove 里的默认行为，之前只知道只与preventDefault()有关。EventTarget.addEventListener()的第三个参数，之前我一直以为就是一个布尔值，控制事件是在冒泡阶段还是捕获阶段触发，不知道还可以接受一个对象配置监听器的相关属性。譬如，passive 设置为 true 时，表示监听器永远不会调用 preventDefault()。详情可见：EventTarget.addEventListener() - Web API 接口参考 | MDN。

后面再看看代码，其实是一些细节上的问题，因为需要判断滚动到顶部和滚动到底部的时候禁止滚动的逻辑存在一些问题，触发了 body 的滚动。这样看来现在这个旧方法是不能用在需要滚动的弹窗上了。而且 touchmove 禁用默认事件的方法，对于安卓设备似乎并不通用，所以还得继续完善。

# 解决滑动穿透的方法

解决滑动穿透除了阻止 touchmove 的默认行为之外，在网上还可以看到许多解决方法。事实上，之前用的基础库里面的方法，应该参照了body-scroll-lock这个库中的处理方式。不过之所以不直接用 body-scroll-lock，也是因为他这里处理得还不够好，在其中只对 IOS 做了处理，非安卓设备只设置了overflow: hidden，而对于安卓设备这样设置恐怕恐怕完全不行。

在之后的探索中，我发现并不是页面所有类似的弹窗都有穿透的毛病，比如选择商品去下单的这个弹窗就没有。原来这个弹窗除了阻止 body 的 touchmove 的默认行为之外，还额外做了一些操作。这个方法是将 body 设置为 position: fixed，禁用 body 的滚动条。因为position: fixed会使滚动位置丢失，所以在设置之前还需要保存当前 body 的 position 的状态到 window 上，以及目前页面滚动的距离。允许滚动的时候，再根据之前记录下来的定位数据，将锁定 body 之前的定位还原回去。

在 tua-body-scroll-lock 中，分别针对安卓、IOS、PC 三端采用不同的方式解决滑动穿透，对于安卓设备就是使用的position: fixed锁定 body 的方法。代码大致如下所示，需要锁定body时，调用setOverflowHiddenMobile，再借助闭包把相关的位置信息存在返回的函数里，等到需要还原 body 时在调用这个返回的函数。这样做的好处是不需要担心变量的冲突问题，可以同时处理多个弹框：

var setOverflowHiddenMobile = function setOverflowHiddenMobile() {
    var $html = $('html');
    var $body = $('body');
    var scrollTop = $html.scrollTop || $body.scrollTop;
    var htmlStyle = Object.assign({}, $html.style);
    var bodyStyle = Object.assign({}, $body.style);
    $html.style.height = '100%';
    $html.style.overflow = 'hidden';
    $body.style.top = "-".concat(scrollTop, "px");
    $body.style.width = '100%';
    $body.style.height = 'auto';
    $body.style.position = 'fixed';
    $body.style.overflow = 'hidden';
    return function () {
        $html.style.height = htmlStyle.height || '';
        $html.style.overflow = htmlStyle.overflow || '';
        ['top', 'width', 'height', 'overflow', 'position'].forEach(function (x) {
        $body.style[x] = bodyStyle[x] || '';
        });
        window.scrollTo(0, scrollTop);
    };
};

对于 IOS 设备，使用的也是阻止 body 的 touchmove 的默认行为的方法。不过如果遇到弹窗内容需要滚动的情况，这里会监听弹窗中的 touchmove 事件，判断到用户滑动到弹窗边缘的时候，才会阻止弹窗的 touchmove 事件的默认行为，从而防止了在弹窗内部滚动到尽头时的穿透问题：

var handleScroll = function handleScroll(event, targetElement) {
  if (targetElement) {
    var scrollTop = targetElement.scrollTop,
        scrollLeft = targetElement.scrollLeft,
        scrollWidth = targetElement.scrollWidth,
        scrollHeight = targetElement.scrollHeight,
        clientWidth = targetElement.clientWidth,
        clientHeight = targetElement.clientHeight;
    var clientX = event.targetTouches[0].clientX - initialClientX;
    var clientY = event.targetTouches[0].clientY - initialClientY;
    var isVertical = Math.abs(clientY) > Math.abs(clientX);
    var isOnTop = clientY > 0 && scrollTop === 0;
    var isOnLeft = clientX > 0 && scrollLeft === 0;
    var isOnRight = clientX < 0 && scrollLeft + clientWidth + 1 >= scrollWidth;
    var isOnBottom = clientY < 0 && scrollTop + clientHeight + 1 >= scrollHeight;

    if (isVertical && (isOnTop || isOnBottom) || !isVertical && (isOnLeft || isOnRight)) {
      return preventDefault(event);
    }
  }

  event.stopPropagation();
  return true;
};
