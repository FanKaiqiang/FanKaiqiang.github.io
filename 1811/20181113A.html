# 注册功能的制作

之前的课上讲过使用AJAX进行浏览器与服务器的交流，这一次就实际的把它运用到这次课上。这次课上是围绕这制作一个登录加注册功能来讲的，首先要先做出注册功能，因为要先注册了账号，才有后面的登录。接下来就先一步步地讲解注册功能的实现。

首先要写出一个注册的界面，大概就是一个form表单，里面放好几个输入框，分别用来接收邮箱、密码、确认密码，还有提交按钮。写好了CSS之后就开始写JS。监听提交按钮的submit事件，并组织form表单的默认事件，防止页面刷新：

let $form = $('#signUpForm')
$form.on('submit', (e)=>{
    e.preventDefault()
    let hash = {}
    let need = ['email', 'password', 'password_confirmation']
    need.forEach((name)=>{
        let value = $form.find(`[name=${name}]`).val()      //  查询符合选择器的值
        hash[name] = value      //将值放入哈希对象中
})

浏览器发送请求后，服务器需要获取请求中的第四部分，因为第四部分就是这次传输的用户注册信息。因为用户上传数据并不是一次性完成的，所以服务器需要监听data事件，将每次获取到的结果推入一个数组结果中，然后再对这个对象进行解析，获取请求发来的数据：

function readBody(request){
    return new Promise((resolve, reject)=>{
        let body = []
        request.on('data', (chunk) => {
        body.push(chunk);
        }).on('end', () => {
        body = Buffer.concat(body).toString();
        resolve(body)
        })
    })

readBody(request).then((body)=>{
    let strings = body.split('&')       // ['email=1', 'password=2', 'password_confirmation=3']
    let hash = {}

        // string == 'email=1'
        let parts = string.split('=')       // ['email', '1']
        let key = parts[0]
        let value = parts[1]
        hash[key] = decodeURIComponent(value)       // hash['email'] = '1'

在获取到刚才的字符串body之后，通过split()方法，把字符串body中以“&”隔开的内容作为字符串分别存入一个数组中，这样的信息就相当于被提炼了一次。然后再对这个数组进行遍历，把数组中的每条字符串再用split()方法做一次“提炼”，这样就拿到了需要的具体的值，将其使用decodeURIComponent()方法转换之后推入对象hash中。

之所以要使用decodeURIComponent()方法，是因为HTTP中的“@”会被转换为“%40”，如果不进行转换的话会干扰接下来的流程。最后let {email, password, password_confirmation} = hash这一句是ES6的语法，它相当于将hash中的每个key作为变量，再将value的值赋给这个变量相当于：

let email = hash['email']
let password = hash['password']
let password_confirmation = hash['password_confirmation']

然后服务器还要进行一些判断，比如说邮箱名是否合法，以及两次密码输入是否一致之类，如果出现问题，服务器就会返回400给请求方，并给出相应的响应信息。为什么浏览器做过判断之后服务器还要进行判断？是因为除了可以通过浏览器可以发请求，使用curl命令也能发请求，如果在服务器上面不加入判断，就可能会接受到一些奇怪的数据。所以服务器也需要做出如下判断：

if(email.indexOf('@') === -1){

}else if(password !== password_confirmation){

# 登录功能的制作

完成了一个帐号的注册之后就要着手登录功能的制作了。至于登录界面只需要将注册页面稍微进行修改就可以了，其他的关于登录信息的判断，大致也可以照抄注册信息的判断流程。所以这里就把登录功能的制作过程中一些中一些重要的部分在这里提一下。

首先将登录信息作为请求内容发出，同样需要像刚才那样服务器需要对接受到的请求内容进行解析。将登录信息提取出来之后，就要判断是否用户名存在数据库中，以及用户名与密码是否匹配：

let found
for(let i=0;i<users.length; i++){
  if(users[i].email === email && users[i].password === password){
    found = true
    break
  }
}
if(found){
  response.setHeader('Set-Cookie', `sign_in_email=${email}`)
  response.statusCode = 200
}else{
  response.statusCode = 401
}

要实现这个功能就要用到这次博客的主角cookie了，正如上面的代码，在返回响应给请求方时，在响应头中添加了“Set-Cookie”这一项，这就相当于给请求方设定了一条cookie。从此以后每当请求方（相同的源）给服务器发送请求时，都要在请求头中加上cookie的内容。这样就相当于服务器给请求方加上了一层身份验证机制，在首页展示用户信息时，只需要获取到cookie的内容并将之进行解析，就可以得到相应的用户信息了：

# Cookie的一些知识

Cookie，有时也用其复数形式 Cookies，指某些网站为了辨别用户身份、进行 session 跟踪而储存在用户本地终端上的数据（通常经过加密）。前文也大概提到了Cookie是什么，在这里先对cookie的特点做一些归纳：

  1. 服务器通过Set-Cookie响应头设置Cookie
  2. 浏览器得到Cookie后，在以后的每次请求中都会带上Cookie
  3. 服务器读取Cookie就知道登录用户的信息

还有几点关于Cookie的问题：

  1. 在Chrome登录，在Safari会有Cookie吗？ 没有
  2. Cookie存在哪里？ 存在C盘中的某个文件
  3. Cookie可以修改吗？ 在开发者工具里就可以修改

在开发者工具里的Application-Cookie-value里就可以直接对刚才的Cookie的内容进行修改，这就可以回到刚才的问题，Cookie其实是不安全的，如果不加上刚才的判断，就可以随便显示想要显示的信息在首页上。

除此之外，Cookie是有有效时间的，可以自行设置，默认值与浏览器有关。同样的，Cookie还可以设置存放时间、针对域名、路径、必须使用https、Httponly（仅http允许访问）等等。
