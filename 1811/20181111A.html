# 文件系统作为数据库

在的网页开发中，有的时候是需要对数据进行操作的，这就涉及到数据库了。数据库有很多种，这次将使用文件系统作为数据库。文件系统支持增删改查，能够长久地进行数据存储。那么使用文件作为数据库，应该如何对数据进行操作呢？下面是一个例子：

var db = fs.readFileSync('.db','utf-8')
1

通过上面的操作，可以将文件“db”中的内容作为字符串以utf-8的格式读取出来，这样就实现了从数据库中读取数据。

fs.writeFileSync('.db',newAmount)
1

这个例子可以将文件“db”的内容替换为newAmount字符串的内容，这样就实现了从数据库中存储数据。

这些操作都是在后端的服务器上完成的，前端的浏览器需要通过发送请求给服务器，服务器再通过对相应的数据进行操作，然后返回响应信息给请求方。那么如何才能实现发送请求呢？

# 使用标签发请求

# <font>发请求

使用<font>标签可以发送请求，比如：

<form action="/pay" method="post">
1

这样在<form>表单提交的时候就会发送POST请求， <form>表单只要提交了，不管是请求与失败，一定会刷新当前页面。阻止默认事件可以防止页面的刷新，但是这样就实现不了数据库传来的信息的即时更新了。通过<iframe>标签可以实现不刷新页面获取数据更新，这样的话，<form>表单提交后的结果会显示在<iframe>中：

<form action="/pay" method="post" target="result">
<iframe name="result" src="about:blank" ...>
1
2

# <img>发请求

使用<img>也可以发请求。使用JavaScript可以创建<img>标签，而浏览器一旦发现在内存中创建了<img>标签就会去请求<img>的src。这时候如果请求成功，就可以获得图片格式的响应信息。这种方法的缺陷是只能设置GET请求，不能设置POST请求。下面是使用<img>发送请求的一个例子：

button.addEventListener((e)=>{
    let image = document.createElement('img')
    image.src = '/pay'
    image.onload = (()=>{})
    image.onerror = (()=>{})
})

通过点击按钮创建<img>标签，这时浏览器就会发起一个请求，通过onload()与onerror()方法的执行情况来判断图片是否请求成功。

# <script>发请求

实际上，所有具有src属性的标签在创建的时候都是回向src指向的链接发送请求的。<script>也一样，但是由于<script>标签的特性，获取到的信息将不会是像<img>标签一样只能是图片，可以是一段JavaScript的代码，这就带来了非常多的可能性。

与添加<img>标签不同，需要将<script>标签放到页面中浏览器才会发送请求：

button.addEventListener((e)=>{
    let script = document.createElement('script')
    script.src = '/pay'
    script.onload = (()=>{
        e.currentTarget.remove()
    })
    script.onerror = (()=>{})
})

使用这种方法创建<script>标签，获取到的<script>标签中的内容是会被执行的，并且是先执行<script>标签中的内容，再执行onload()方法。onload()方法的触发一定要保证<script>标签中的内容正确，就像之前的<img>标签一样，如果返回的内容并不是图片格式的信息，也会触发onerror()方法。之所以<script>标签中的内容会执行，是因为其内容是基于HTTP协议设定为JavaScript格式，并且次标签确实被置于<script>标签中。

上面的例子还实现了在每次<script>标签创建成功后删除该<script>标签，这样在页面上就看不到这个<script>标签的创建过程了，因为这个<script>标签一被创建、执行完毕之后就被删除了。

下面总结一下使用创建<script>标签发送请求的完整过程：

  1. 用户点击按钮触发点击事件；
  2. 创建<script>标签，其中src属性为请求路径；
  3. 将<script>标签放到页面里；
  4. 浏览器根据路径发起GET请求
  5. GET成功，首先会执行服务器返回的<script>标签的内容
  6. 根据执行的内容局部更新源码内容
  7. 执行完毕后监听是否请求创建<script>标签成功（或失败）
  8. 删除刚才创建的<script>标签

使用这种方法发送请求是不受域名限制的，任何一个网站都可以使用另一网站的js，这时候请求路径src就要设置为另一个网站的地址了。

# JSONP

上述的方法代码耦合性过高。因为如果使用这种方法，后端服务器需要深入了解前端页面的内容，才能返回前端需要执行的JavaScript代码。需要对上述方法进行完善，使得后端只需要调用前端写好的函数，而不是实际操作JavaScript代码。

下面是一个示例，前端在<script>标签的src属性中额外传递了一个函数名信息给服务器，服务器取得函数名之后将前端请求的数据作为这个函数的参数，最后将函数的调用作为<script>标签的内容返回给前端。这样前后端的职责就得到了很好的区隔，再没有出现需要在后端服务器写前端代码的情况了：

请求方：

button.addEventListener('click', (e) => {
    let script = document.createElement('script')
    let functionName = 'frank' + parseInt(Math.random() + 100000, 10) //为了防止产生重复的函数名，使用随机数
    window[functionName] = function (result) {
        alert(result)
    }
    script.src = 'jack.com:8002/?callback=' + functionName
    document.body.appendChild(script)
    script.onload = function (e) {
        e.currentTarget.remove()
        delete window[functionName]
    }
    script.onerror = function () {
        alert('fail')
        e.currentTarget.remove()
        delete window[functionName]
    }
}

如果上面服务器给函数传入的参数是一个符合JSON语法的对象，那么上面的内容就是一个JSONP的完整过程，再来归纳一下：

  1. 请求方动态创建script标签，以src指向查询响应方，同时传一个查询参数callback
  2. 响应方根据查询参数callback，构造形如xxx.call(undefined,'数据')的响应
  3. 浏览器接受响应，并执行xxx.call(undefined,'数据')
  4. 请求方获取到了需要的数据

最后来看一下百度百科上关于JSONP的介绍，自此，对于JSONP总算是有一种豁然开朗的感觉：

JSONP(JSON with Padding)是JSON的一种“使用模式”，可用于解决主流浏览器的跨域数据访问的问题。由于同源策略，一般来说位于 server1.example.com 的网页无法与不是 server1.example.com的服务器沟通，而 HTML 的<script> 元素是一个例外。利用 <script> 元素的这个开放策略，网页可以得到从其他来源动态产生的 JSON 资料，而这种使用模式就是所谓的 JSONP。用 JSONP 得到的资料并不是 JSON，而是任意的JavaScript，用 JavaScript 直译器执行而不是用 JSON 解析器解析。

![JSONP](20181111A/003.jpg)

正如上面所说的，使用这种方法可以获取JSON信息，但是是以JavaScript格式来读取的。因此JSONP是解决网站之间的跨域读取数据的一种方案，与JSON关系并不太大。

此外，JSONP可以使用jQuery实现，下面是一个例子：

 $.ajax({
 url: "http://jack.com:8002/pay",
 dataType: "jsonp",
 success: function( response ) {
     alert(response)
     }
 }
 })

最后要提一点，JSONP是不支持POST请求的，原因概括起来可以这样说：

  1. 因为JSONP是通过动态创建script实现的；
  2. 动态创建script的时候，只能用get，没有办法用post；
