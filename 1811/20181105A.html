【整理】DOM API

广宇 2018-11-6 JavaScript DOM

今天下午想了半天博客该写些什么，但是总感觉这几天学的比如说轮播啊什么的都不太好写博客。昨天我的打算是每晚都是用来写博客的，因为我的打算是每天完成一个系统课任务，太多了感觉圆消化不了。

刚才也一直在会看我自己的笔记，发现简直不能看，所以还是要整理一下。现在做一个规定，本星期的博客就整理上星期所学的知识，希望这样也能起到一些复习作用。

这次的博客的主题还是DOM，上次写过一篇关于DOM的博客，但是之前因为系统课还没学到DOM这节，所以写起来也只是一知半解，这次的话树妖目的就是把我笔记上的东西捋一捋，毕竟笔记做得有点难看，如果不整理出来笔记就报废了。所以这节课就先不去查资料扩展知识了，尽可能地把我那次课的知识复盘一次。

# DOM

DOM的全称是Document Object Model，即文档对象模型。之前的博客引用了MDN的定义，这里就不重复操作了。

「文档对象模型」中的文档指的是XML文档，而HTML是衍生于XML的。那么模型指的又是什么一回事呢？

可以知道HTML的结构大致如下：顶部的html声明、html标签、head标签和body标签。总体来说，html文档是呈现一种树形结构的，那么如何将这个树形结构存放于内存中呢？首先想到的是对象，而将Document（文档）转化为object（对象）的过程，就把它称之为一种模型（Model）的映射。这个过程的全称即是DOM。如图所示，就是由HTML树转化而来的DOM树：

DOM的标准决定了这些对象要以什么方式来存放这个文档。其中存储html文档中的各种元素与内容的对象分别由以下几种构造函数构造，比如ELement（元素）、Text（文本）、Document（根节点）等，正如JS中的对象都是继承于Object的，在页面中的这几种构造函数构造出来的对象都是继承于Node（节点）的。Node是在这些种对象的原型链的顶端的，也可以理解为元素、文本以及根节点在页面中都是属于节点的。

以获取『<span>字符串</span>』标签中的文本为例：

span.firstChild             //"字符串"
typeof span.firstChild      //"object"
1
2

由此可见，span的第一个子元素是对象，并非浏览器展示出来得那样像个字符串。这个对象是个文本节点，原型指向Text.prototype。这里需要指出的是，如果在html文档中有注释，注释也会被当做节点，为注释节点，原型链指向comment.prototype。

所以，页面上所有标签都有雨它对应的构造函数，浏览器只要发现一个标签，就用对象的构造函数给它构造出对应的对象。

# DOM API

DOM的主要功能就是在操作页面中的节点时，可以通过操作对应对象的API来实现。这些相关的API就是DOM API。

Node主要分为Document、Element、Text三类。其中Node也是一种object，作为所有节点对象的原型链顶端，Node的API是所有类型的节点都可以使用的。下面选择性地对一些API进行讲解。

# Node的属性

在学习API的过程中，常常会见到这些单词，很多API都是由这些单词拼凑而成的，知道这些单词的意思可以帮助又快又好地理解这些API的含义：

  • child、children、parent 子节点、父节点
  • node 节点
  • first、last 首个、末个
  • next、previous 下一个、上一个
  • type 类型
  • value、text、content 值、文本、内容
  • inner、outer 里面、外面
  • element 元素、标签
  • sibling、siblings 兄弟元素

下面就将几个由这几个单词拼凑起来的几个具有代表性的Node的属性。

  • ChildNodes ：顾名思义，即是所有子节点。其中要注意，获取到的文档中的回车也算是子节点。因此所有子节点并不等于所有子元素。
  • Children：所有子元素。其中不包含文本节点（回车等）。
  • firstChild：首个子节点。lastChild就是末个子节点。
  • firstElementChild：首个子元素。
  • previousSibling：前一个兄弟节点。nextSibling即为下一个兄弟节点。
  • previousElementSibling：前一个兄弟元素。
  • innerText：节点里面的文本。
  • nodeName：节点名。注意，获取的节点名一般都是大写的，比如document.body.nodeName就会返回BODY，但是svg元素除外，svg节点会返回小写的节点名称。
  • nodeType：节点属性。会返回一个代表节点属性的数值，在之前的博客里讲到过。
  • nodeValue：返回节点的值
  • textContent：文本内容
  • innerText：内容文本
  • ……

这里着重讲一下textContent（文本内容）与innerText（内容文本）的区别：

  1. textContent会获取所有元素的内容，包括<script>与<style>，但是innerText不会。
  2. innerText不会返回隐藏文本的内容（display:none），但是textContent会。
  3. innerText会受CSS的影响触发重排，textContent不会。
  4. 在IE中，如果对innerText进行修改，会破坏当前元素及后代的文本节点。

可以看出innerText是会去识别CSS以及JS的内容的，而textContent不会，这是二者的主要区别。

# Node的方法

  • .appendChild()：把某元素设为子元素
  • .cloneNode()：克隆节点。如果采用深拷贝，须传入true，此时后代所有节点都会被克隆。如传入false，则只会克隆该节点本身。
  • .contains()：判断一个元素是否包含另一个元素
  • .hasChildNodes()：判断是否有子节点
  • .insertBefore()：把该节点插在某节点前
  • .isEqualNode()：是否相等节点。节点内容内容相等即可。
  • .isSameNode()：是否相同节点。同一个节点才是相同节点。
  • .removeChild()：在页面中删除节点，不过内存中仍然存在。
  • .replaceChild()：将子元素替换。
  • .normalize()：将当前节点及其后代常规化。规范化后的DOM树中不存在空文本与相邻的两个文本节点。
  • ……

# Document的属性

  • anchors：锚点。该属性会返回那些具有name属性的a标签
  • body：获取body标签
  • characterSet：使用的字符集
  • childElementCount：子元素个数
  • documentElement：html元素（根元素）
  • domain：域名
  • fullscreen：全屏
  • head：获取head元素
  • hidden：返回是否被隐藏
  • images：返回所有img标签
  • link：返回a标签
  • onxxx：事件监听
  • plugin：插件
  • referer：源页面。通过这个属性可以形成防盗链。
  • script：获取所有script标签
  • scrollingElement：正在滚动的标签
  • visibilityState：页面是否被显示
  • ……

# Document的方法

  • .close()：创建元素
  • .createElement()：创建文本节点
  • .createTextNode()：退出全屏
  • .exitFullScreen()：退出全屏
  • .execCommand()：执行命令
  • .getSelection()：获取选中文本
  • .hasFocus()：判断用户是否在当前页面
  • .querySelector()：选择器选择到的第一个元素
  • .querySelectAll()：选择器选择的所有元素
  • .write()：写入内容
  • ……

事实上DON API 数不胜数，在课上也就讲到这里就完结了，Element的API都还没有讲。如果全都要讲，几篇博客都讲不完。所以这节课的主要目的还是知道DOM是怎么一回事，DOM API是怎么样的机制、如何使用，在日后的学习过程中见到相关的内容能够游刃有余。

如何学习API？DOM API无外乎与节点相关的增删改查，要想学好API。首先要搞清楚英文的意思，如果弄明白了API的英文意思也就学会了一半。除此之外还可以看MDN，可以看到，API的总数是相当之多，今天罗列的只是沧海一粟，如果遇到什么不明白的API，往这里查就是了：Web API 接口 | MDN
