# 闭包很重要

好了步入正题。这次的主题是闭包，关于闭包早在开始学JS的时候就接触了:

「函数」和「函数内部能访问到的变量」（也叫环境）的总和，就是一个闭包。

闭包看起来不是那么难的概念，正如下面MDN中的例子所示，里面的 name 变量与 displayName 函数就形成了一个闭包：

function init() {
    var name = "Mozilla";       // name 是一个被 init 创建的局部变量
    function displayName() {    // displayName() 是内部函数,一个闭包
        alert(name);            // 使用了父函数中声明的变量
    }
    displayName();
}
init();

# 闭包是什么？

回到刚才的那个例子：

function init() {
    var name = "Mozilla";       // name 是一个被 init 创建的局部变量
    function displayName() {    // displayName() 是内部函数,一个闭包
        alert(name);            // 使用了父函数中声明的变量
    }
    displayName();
}
init();

在这个例子中，init() 创建了一个局部变量 name 和一个名为 displayName() 的函数。displayName() 是定义在 init() 里的内部函数，仅在该函数体内可用。

运行这段代码可以发现， displayName() 内的 alert() 语句成功的显示了在其父函数中声明的 name 变量的值。这说明虽然 displayName() 内没有自己的局部变量，但是它可以访问到外部函数的变量name 。

这个函数作用域的例子介绍了引擎是如何解析函数嵌套中的变量的。函数作用域中使用的域，是变量在代码中声明的位置所决定的。嵌套的函数可以访问在其外部声明的变量。但是，如果有同名变量 name 在 displayName() 中被定义，则会使用 displayName() 中定义的 name 。

上文提到过这个 name 变量与 displayName 函数就形成了一个闭包，但是平时看到的闭包是会return一个函数的。把这个例子修改一下，看看闭包究竟是怎么一回事：

function makeFunc() {
    var name = "Mozilla";
    function displayName() {
        alert(name);
    }
    return displayName;
}

var myFunc = makeFunc();
myFunc();

运行这段代码和之前的 init() 示例的效果完全一样。其中的不同 — 也是有意思的地方 — 在于内部函数 displayName() 在执行前，被外部函数返回。

第一眼看上去，也许不能直观的看出这段代码能够正常运行。一旦 makeFunc() 执行完毕，可能会认为 myFunc() 不能访问到 name 变量。但是代码运行得并没问题。这就是因为JavaScript中的函数会形成闭包。

刚才说了：

「函数」和「函数内部能访问到的变量」（也叫环境）的总和，就是一个闭包。

上面的这个环境包含了这个闭包创建时所能访问的所有局部变量。在例子中，myFunc 是执行 makeFunc 函数后返回的 displayName 函数实例，而 displayName 实例仍可访问这个环境中的变量，即可以访问到 name 。由此，当 myFunc 被调用时，name 仍可被访问，其值 Mozilla 就被传递到alert中。

再来说说为什么要return。因为如果不 return，你就无法使用这个闭包。把 return displayName 改成 window.displayName = displayName 也是一样的，只要让外面可以访问到这个 displayName 函数就行了。

所以 return displayName 只是为了 displayName 能被使用，也跟闭包无关。

再来看一个例子，

function makeAdder(x) {
  return function(y) {
    return x + y;
  };
}

var add5 = makeAdder(5);
var add10 = makeAdder(10);

console.log(add5(2));  // 7
console.log(add10(2)); // 12

在这个示例中，定义了 makeAdder(x) 函数，它接受一个参数 x ，并返回一个新的函数。返回的函数接受一个参数 y，并返回x+y的值。创建了两个新函数 — 一个将其参数和 5 求和，另一个和 10 求和。

add5 和 add10 都是闭包。它们共享相同的函数定义，但是保存了不同的词法环境。在 add5 的环境中，x 为 5。而在 add10 中，x 则为 10。

所以是不是可以这样考虑，当一个变量接受一个形成闭包的赋值时，这个变量不仅收到了这个函数的内容，连带的这个函数的「函数内部能访问到的变量」也一并被赋予了。

result.add()        //4401

test()函数return的是一个对象，里面有一个数值n，与一个存在闭包的函数add()。分别把同一个对象赋给两个变量，就这个对象中的函数add()而言，尽管赋值时收到的是相同的函数与闭包环境，但是因为这两个变量在内存中存放在不同的位置，所以这两个变量中的add()函数的并不是在同一个闭包环境。

所以执行了result.add()两遍之后，add()的闭包环境中的变量n的值也加了两遍，就是4401。那为什么result.n仍然是4399呢？那是因为加的是result.add()的闭包环境的n，并不是result这个对象里面的n，result.n的值一直还保持着test()最初赋值时候的值。

因为result与result2是在两个不同的闭包环境，所以result2.add()执行后加的是result2.add()的闭包环境的n，加完之后是4400。这就是关于这道题解决的全部过程。

# 闭包的作用

可以发现，所演示的闭包都是套在函数里的。不过在刚才定义的闭包里面并没有提到外面要套一个函数。这是因为需要使用局部变量，如果不在外面套一层函数，里面的变量就成了全局变量，就达不到使用闭包的目的了。所以函数套函数只是为了造出一个局部变量，跟闭包无关。

那么来讲讲一个关于闭包的一个典型的运用：创建模块。将操作函数暴露给外部，而细节隐藏在模块内部。比如下面的例子：

function addSize() {
    var size = 14
    return function() {
        size++
        document.body.style.fontSize = size + 'px';
        console.log(size)
    }
}

var size1 = addSize()
size1()

顺便看看一个关于闭包的常见错误：

function showHelp(help) {
    document.getElementById('help').innerHTML = help;
}

function setupHelp() {

      {'id': 'email', 'help': 'Your e-mail address'},
      {'id': 'name', 'help': 'Your full name'},
      {'id': 'age', 'help': 'Your age (you must be over 16)'}
    ];

  for (var i = 0; i < helpText.length; i++) {
    var item = helpText[i];
    document.getElementById(item.id).onfocus = function() {
      showHelp(item.help);

}

运行这段代码后，您会发现它没有达到想要的效果。无论焦点在哪个input上，显示的都是关于年龄的信息。

原因是赋值给 onfocus 的是闭包。这些闭包是由他们的函数定义和在 setupHelp 作用域中捕获的环境所组成的。这三个闭包在循环中被创建，但他们共享了同一个闭包环境，在这个环境中存在一个变量item。当onfocus的回调执行时，item.help的值被决定。由于循环在事件触发之前早已执行完毕，变量对象item（被三个闭包所共享）已经指向了helpText的最后一项。
