# 1、说说盒模型（必考）

盒模型的四个组成结构：content（内容）、padding（内边距）、border（边框）、margin（外边距）

盒模型都具备这些属性。这些属性可以把它转移到日常生活中的盒子（箱子）上来理解，日常生活中所见的盒子也就是能装东西的一种箱子，也具有这些属性，所以叫它盒模型。

主要有两种盒模型，W3C盒模型是与IE盒模型。区别就是对宽高的定义不同。：

content-box(W3C盒模型)：width = content(内容区宽度)

border-box(IE盒模型)：width = content + padding-left + padding-right + border-left + border-right

默认使用标准的w3c盒子模型，如果需要使用IE的盒子模型，可以使用box-sizing属性进行修改：box-sizing:border-box。

# 2、CSS reset 和 normalize.css 有何区别？

这道题其实是在考英文，因为reset就是重置的意思，normalize的意思则是标准化，区别已经很明显了。

CSS Reset，意为重置默认样式。HTML中绝大部分标签元素在网页显示中都有一个默认属性值，通常为了避免重复定义元素样式，需要进行重置默认样式（CSS Reset），即抛弃原有的默认样式。

Normalize.css是一种CSS reset的替代方案。Normalize.css能够让所有浏览器的标签都跟标准规定的默认样式一，使得页面在各浏览器上显示的标签的默认样式基本统一。相比于传统的CSS reset，Normalize.css是一种现代的、为HTML5准备的优质替代方案。

# 3、如何居中（必考）

这一点主要还是看平时的总结了，居中的方式挺多的。如果是水平居中的话：

  • 内联元素：给父元素加text-align: center;
  • 块级元素：margin: 0 auto;

# (2) 选择器的优先级

选择器的优先级遵循一下几个原则：

  1. 选择器越具体优先级越高
  2. 相同优先级，写在后面的选择器会覆盖前面的选择器
  3. important优先级最高，但一般不用

具体来说，各选择器的级别是这样排列的：

  1. 在属性后面使用 !important 会覆盖页面内任何位置定义的元素样式。
  2. 作为style属性写在元素内的样式
  3. id选择器
  4. 类选择器
  5. 标签选择器
  6. 通配符选择器（*）
  7. 浏览器自定义或继承

# 5、BFC是什么？

BFC（Block Formatting Context）即“块级格式化上下文”，BFC是W3C CSS 2.1 规范中的一个概念，它决定了元素如何对其内容进行定位，以及与其他元素的关系和相互作用。当涉及到可视化布局的时候，BFC提供了一个环境，HTML元素在这个环境中按照一定规则进行布局。一个环境中的元素不会影响到其它环境中的布局。比如浮动元素会形成BFC，浮动元素内部子元素的主要受该浮动元素影响，两个浮动元素之间是互不影响的。也可以说BFC就是一个作用范围。

之前写过一篇关于BFC的博客：【归纳】BFC解惑，这里就不赘述了。关于BFC的面试回答主要也是要侧重在作用方面，因为概念上的东西说不清楚，下面是两个例子：

  • overflow:hidden 清除浮动。（用 .clearfix 清除浮动更好）
  • overflow:hidden 取消父子 margin 合并。http://jsbin.com/conulod/1/edit?html,css,js,output （设置 padding-top: 1px;也可以达到相同效果）

# 6、如何清除浮动？

这里就说两种清除浮动的方法，已经足够回答面试了：

# overflow:hidden

借助BFC的特性清除浮动，但是不推荐使用。

# .clearfix 清除浮动

.clearfix 清除浮动的代码如下：

.clearfix::after{
  content:'';
  display:block;
  clear:both;
}
/*兼容IE*/
.clearfix{
  zoom:1;
}
