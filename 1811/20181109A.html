【归纳】DOM事件模型

广宇 2018-11-10 JavaScript DOM

今晚的话继续写博客，这几天决心下来专心学系统课，还是把系统课给看完再来考虑算法的事吧，不然感觉有点力不从心。这次的博客主题是DOM事件模型，之前写过几篇DOM的博客，这回的博客就专门就DOM事件模型（DOM Event Model）进行讨论。

这也是上个星期学的内容了，一个星期没看，果然是生疏不少，还是得把笔记整理一下。

# DOM 0 的事件监听

DOM使JavaScript有能力对HTML上的事件做出反应。这些事件包括鼠标键盘的点击事件、移动事件以及页面中内容的变化等。HTML元素事件是浏览器内在自动产生的,当有事件发生时html元素会向外界(这里主要指元素事件的订阅者)发出各种事件,如click,onmouseover,onmouseout等等。

JavaScript监听事件主要有三种方式，首先简单地讲一下用HTML 属性监听的方法，比如：

<button onclick="alert('Hello world!')">
1

点击这个按钮，就会触发点击事件。但是应该尽量避免这种书写方式，因为这将造成内容/结构及行为不能很好的分开，使得HTML文档变得臃肿并减少可读性，而且其中的bug会很难找到。

如果不用这种方法，还有其他两种方法可以用，分别是DOM 0版本与DOM 2版本的的事件监听方法。先来看看DOM 0版本的，DOM 0版本即是在DOM标准发布前就已经存在的事实规范，因此被称为DOM 0。而后面的DOM 1才是第一个标准，主要是DOM 0的汇总。

下面是一个按钮点击事件的监听：

document.getElementById('click').onclick = function(event){
    console.log(点击我了);
}

3

从上面的代码可以知道，一旦用户点击id为“click”的这个标签，就会触发上面函数中的代码。那么onclick是到底什么呢，这其中的过程又是怎么样的呢？从这里就可以很容易看出，实际上onclik是这个id为“click”的标签的一个属性，这个属性的内容是一个函数对象，如果要解除这个点击事件，把这个属性赋值为null就可以了。

但是如果在点击这个标签时要执行多个函数该怎么办的，如果是像这样：

var click = document.getElementById('click');
click.onclick = function(){
    console.log('第一条函数');
};
click.onclick = function(){
    console.log('第二条函数')
}

因为onclick只是click的一条属性，所以后面设定的onclick属性值会覆盖在前面设定的值，这样是做不到在点击事件触发时执行两个函数的。因此上面会只得到“第二条函数”的输出。这个时候就要使用DOM 2的事件监听的方法了。

# DOM 2 的事件监听

下面就是经常见到的事件监听的一种方法：

myButton.addEventListener('click', function(){
    alert('Hello world');
}, false);
1
2
3

这就是DOM 2 的事件监听，在实现的效果上与DOM 0的方法并没有差异。其中的区别在于，DOM 0 中的onclick是一个标签的属性，是惟一的，做不到在一个属性上绑定两个点击事件，如果存在两个onclick属性的赋值就会产生冲突，这一点刚才也提到了。

为了解决这个问题才有了DOM 2 的事件监听的方法。当为一个标签添加多个事件处理函数时，相当于给一个队列添加事件处理函数，是以先进先出的顺序执行的。

那么问题来了,比如点击一个嵌套的标签。如下图所示：

document.addEventListener('click',f1)
document.documentElement.addEventListener('click',f2)
document.body.addEventListener('click',f3)
clickDiv.addEventListener('click',f4)

# DOM 事件模型

从上面的图可以看出，事件的执行顺序无外乎两种，从内到外或者从外到内。这两种事件的执行顺序就是DOM的事件模型。

事件模型描述的是从页面中接收事件的顺序。DOM事件模型分为两类：一类是IE所使用的冒泡型事件（Bubbling）；另一类是DOM标准定义的冒泡型与捕获型（Capture）的事件。除IE外的其他浏览器都支持标准的DOM事件处理模型。

造成这种现象的原因是IE和网景团队提出了相反的事件模型概念。IE事件流是事件冒泡流，而网景是事件捕获流。

# 事件冒泡

IE的事件流叫做事件冒泡，即事件开始由最具体的元素接收，然后逐级向上传播到较为不具体的节点。如上图所示，4、5、6、7就是事件冒泡阶段。

点击元素会从最具体的元素向上传播。所有现代浏览器都支持事件冒泡，但是具体实现还是有些差别，IE5.5及更早版本的事件会跳过<html>元素。如果的事件处理函数设定在冒泡阶段执行，以前文的例子为例，执行的顺序就是f4、f3、f2、f1。DOM 0级添加的事件处理函数也会在事件模型的冒泡阶段被处理。

# 事件捕获

网景公司提出的另一个事件流叫做事件捕获，事件捕获的思想上不太具体的节点应该更早接收到事件，而最具体的元素应该最后接收到事件，事件捕获的目的的事件到达预期目标之前捕获它。之前的例子来说顺序就是f1、f2、f3、f4。

由于老版的浏览器不支持事件捕获，所以一般使用的都是事件冒泡。在DOM 2中又该怎么设置事件处理函数是在冒泡阶段还是捕获阶段处理呢？再来看一下DOM 2事件监听的格式：

myButton.addEventListener('click', function(){
    alert('Hello world');
}, false);
1
2
3

里面有一个奇怪的布尔值参数，这个参数就是用于指定这个事件处理函数是在哪个阶段执行的。如果传入的值为false，函数就在冒泡阶段执行；如果传入的值为true，函数就在捕获阶段执行。jQuery不支持传入这个参数来设定冒泡或者捕获。

DOM2级规定的事件模型实际上包含三个阶段，事件捕获阶段、处于目标阶段和事件冒泡阶段。因此会先执行完捕获阶段的函数，再执行冒泡阶段的函数。就当前点击的这个标签而言，其中的事件处理函数其实都是在「处于目标阶段」执行的，并没有冒泡和捕获的区分，以默认的先进先出的队列顺序来执行。

# 阻止冒泡

以这节课的课后习题为例，来讲解一些阻止冒泡的方法。这此的习题实现的是一个点击按钮显示浮层，然后点击浮层的外部就会使浮层消失，大概是这样的一个效果：http://js.jirengu.com/zejon/2/edit?html,css,js,output

假设为button添加一个点击事件处理函数f1，f1可以展开浮层，给document添加一个点击事件处理函数f2，f2可以隐藏浮层。按照冒泡阶段的事件执行顺序，会先执行f1再执行f2，如果不对冒泡进行阻止，隐藏方法就会与显示方法合并，使得无法实现浮层展示的效果，所以要在f1中加上阻止冒泡：

$(click).on('click',(e)=>{
    $(popover).show()
    e.stopPropagation()     //阻止冒泡
})

3
4

这样在执行完函数f1后，冒泡阶段就被终止了，这样达到不执行f2的效果了。除此之外还有一种方法，可以做到将f2的内容过一会再添加到document：

$(document).on('click',function f2(){
    setTimeout(()=>{
        $(popover).hide()
    },0)
})

上面的代码可以实现，在冒泡阶段结束0s后把f2的内容添加到document，这样f1就能顺利执行而不受到f2的干扰了。
