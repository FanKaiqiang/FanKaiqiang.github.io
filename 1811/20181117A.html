【押题】HTML面试知识

广宇 2018-11-18 HTML 押题

到今天为止系统班的课程其实以及看得差不多了，今天早上我看了一下面试技巧以及简历制作技巧，不过因为我现在毕设也还没开始做，而且我目前的水准还没马上到找工作的时候，所以这两节课我不是今天就能把它刷完的。

简历的话我打算花点时间打磨，面试的知识我今天也不打算一次就看完，贪多嚼不烂。我还是把所有面试需要了解的知识分做几次在博客上做些归纳吧。

这次博客的主题是 HTML 有关的面试题，事实上在网上可以找到非常多的关于 HTML 的面试题，所以算是一个押题向的博客。关于面试技巧方面这里也顺带提几点：

  1. 遇到知道但是不知从何下手的题就举例说明
  2. 遇到不会的题目先承认自己不会，再通过追问题目的细节转移焦点，将不会的变为会的
  3. 在回答一些问题的时候可以侃侃而谈，而不是单纯地回答问题

下面就对几个面试可能会遇到的题目用这几种思路来解答一下。

# 1、如何理解 HTML 语义化？

这里提供两种答法，一种是比较正经地把问题答完，另一种是以前端发展的时间为角度来回答这个问题。各有长处，总之最重要的还是把问题搞清楚，希望到时候能记住吧。

# 举例答法

根据内容的语义化，选择合适的标签便于开发者阅读。使得开发者在写出更优雅的代码的同时让浏览器的爬虫和机器很好地解析。比如说，段落用 p 标签、边栏用 aside 标签，主要内容用 main 标签等等。

# 侃侃而谈

以前最开始是 PHP 后端写 HTML ，他们不会 CSS ，于是就用 table 来布局，但是 table 是用来展示表格的，这严重影响了 HTML 的语义化。

后来有了专门写 CSS 的前端，他们会使用用 DIV + CSS，主要是用 float 与绝对定位布局，稍微符合了 HTML 的语义化。

再后来前端专业化了，知道 HTML 各个标签的用法，于是会用恰当的标签来展示内容，而不是全用 div ，而是尽量使用 h1、ul、p、main、header 等标签。

# 2、 meta 的 viewport 是干什么用的，怎么写？

# meta:vp的内容

meta:vp 的内容有点长，这就要死背了：

<meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0" >
1

上面的意思是：

  • 宽度等于设备宽度
  • 用户不准放大缩小
  • 初始状态缩放倍数为1.0
  • 最大缩放倍数为1.0
  • 最小缩放倍数为1.0

作用就是控制页面在移动端不缩小显示。

# 侃侃而谈

一开始所有页面都是做给 PC 端浏览的，乔布斯推出 iPhone3Gs 的时候，页面是不适应手机屏幕的，所以苹果的工程师想了一个办法，默认把手机设置为模拟 980px 宽度，所以页面会缩小显示。

后来只能手机普及了，这个功能在部分网站不需要了，所以用 meta:vp 让手机不要缩小的网页。

# 3、Canvas元素是干什么的？

<canvas> 是 HTML5 新增的元素，可用于通过使用JavaScript中的脚本来绘制图形。例如，它可以用于绘制图形，制作照片，创建动画，甚至可以进行实时视频处理或渲染。
