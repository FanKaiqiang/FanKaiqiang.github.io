这回讲比较重要的快速排序，之前讲的几种排序时间复杂度都是O(n2)，这回的快速排序在实现方式上比之前的排序要复杂一些，但是这个算法既然被称为「快速」排序，那么一定是存在快速之处的。来看看维基百科上关于快排的定义：

快速排序（英语：Quicksort），又称划分交换排序（partition-exchange sort），简称快排，一种排序算法，最早由东尼·霍尔提出。在平均状况下，排序n个项目要 O(nlogn)次比较。在最坏状况下则需要O(n2)次比较，但这种状况并不常见。

老规矩，试着用自然语言描述一下快排的思路，然后再来分析一下快排的实现代码。

快速排序的关键在于使用分治法策略来把一个序列分为两个子序列。假设有一个待排数组，要用快排的方法给它排序

  1. 从数组中选取一个基准数，并遍历这个数组。
  2. 将大于这个基准数的数放在基准数右边，将小于这个基准数的放在基准数左边。这样该基准数就处于数列的中间位置
  3. 将基准数左右两边的数再次执行1、2两步的操作，直到不能再分（长度为一）为止。
  4. 这样得到的数组就是排序完成的数组了。

事实上，快速排序通常明显比其他算法更快，因为它的内部循环（inner loop）可以在大部分的架构上很有效率地达成。下面是一个快速排序的动图示例，与用自然语言描述的方法基本一致：

以下是 JavaScript 版本的的代码实现：

function quickSort(arr) {
  if(arr.length <= 1) {
    return arr
  }
  let leftArr = [] 
  let rightArr = []
  for(let i = 1; i < arr.length; i++) {
    if(arr[i] >= arr[0]) {
      rightArr.push(arr[i])
    } else {
      leftArr.push(arr[i])
    }
  }
  return quickSort(leftArr).concat(arr[0]).concat(quickSort(rightArr))
}

var arr = [10, 34, 21, 47, 3, 28]
quickSort(arr)

对这段代码进行一下分析：

  1. 待排数组arr，如果数组长度小于或等于1(arr.length <= 1)，直接返回数组；
  2. 准备基准数（arr[0]）的左数组与右数组，将小于基准数的数组放入左数组（leftArr.push(arr[i])），大于基准数的数字放入右数组（rightArr.push(arr[i])）；
  3. 将左数组+基准数+右数组拼成一个数组（quickSort(leftArr).concat(arr[0]).concat(quickSort(rightArr))）作为返回值，同时左右数组再次分别投入该算法进行排序

上面quickSort 函数内每次执行新创建两个数组，多次递归后会创建大量数组，在空间上存在"浪费"。可以在原数组上操作：

function quickSort(arr) {
  function _quickSort(arr, start, end) {
    if(start >= end) return
    let key = arr[end]
    let left = start, right = end - 1
    while(left < right) {
      while(arr[left] < key && left < right) left++
      while(arr[right] >= key && left < right) right--
      [arr[left], arr[right]] = [arr[right], arr[left]]
    }
    if(arr[left] >= arr[end]) { 
      [arr[left], arr[end]] = [arr[end], arr[left]]
    } else {  // 如 [2, 1, 3, 4]
      left++
    }
    _quickSort(arr, start, left - 1)
    _quickSort(arr, left + 1, end)
  }
  _quickSort(arr, 0, arr.length - 1)
  return arr
}

这种空间利用率的的方法代码思路大致如下：

  • 对于一个数组，挑选最后一个值作为参考值(key)
  • 从数组的头部开始扫描，如果值比参考值小，继续往后扫描，直到扫描到的值(左值)比参考值大
  • 从数组的尾部(参考值的前一个)开始扫描，如果值比参考值大，继续往前扫描，直到扫描到的值(右值)比参考值小
  • 此时交换扫描停止时的这两个值
  • 继续上面的逻辑，直到左值和右值相遇
  • 如果相遇时的值大于等于参考值，让参考值和相遇值调换位置(一般情况)
  • 如果相遇时的值小于参考值，不调换，但 left 后移以为 (特殊情况，如 [2, 1, 3, 4, 5])

经过上面的处理后，就会把数组变成以原数组末尾数字为分割（左边都比它小，右边都比它大）的数组。然后分别对参考值左侧和右侧通过类似的逻辑继续处理。
