【归纳】JS 中的 Date

广宇 2019-1-24 JavaScript

距离我从广州回家也已经有四五天了，这段时间一直在看毕设课程，拓展课程与博客都没有什么进展，而且毕设也只是看完而已，要想真正把这些内容做成项目，任务还是很重的。

所以回来之后不能再像之前那样懒惰了，每天把自己绷紧一点，留给我的时间已经不多了。

今天的博客内容是 JavaScript 中的 Date，并不是很难的内容。之所以想把这些内容做些归纳的原因是，现在我对 Date 的了解甚少，但是又经常会遇到它们。所以这篇博客也算是我的一个学习记录吧，具体的学习方式依旧是把 MDN 中的相关内容过一遍：Date | MDN，希望这次博客之后我对 Date 就能是了如指掌了吧。

# 一、构造函数 Date()

Date 对象是用来处理日期和时间的对象，调用 Date 构造函数能够实例化日期对象。下面是几种构造 Date 对象的方法：

var today = new Date();
var today = new Date(1453094034000); 
var birthday = new Date('December 17, 1995 03:24:00');
var birthday = new Date('1995-12-17T03:24:00');
var birthday = new Date(1995, 11, 17);
var birthday = new Date(1995, 11, 17, 3, 24, 0);

其中大概能够分为以下几种情况：

  1. 如果没有输入任何参数，则 Date 的构造器会依据系统设置的当前时间来创建一个Date对象。
  2. Date 构造函数接受数值参数，此参数代表自1970年1月1日00:00:00 (世界标准时间) 起经过的毫秒数，据此创建 Date 对象。
  3. Date 构造函数接受字符串参数，此参数应为能被 Date.parse() 方法识别表示日期的字符串值，据此创建 Date 对象。

Date 构造函数具有一个 Date.length 属性，值是 7。这是该构造函数可接受的参数个数，分别代表年、月、日、时、分、秒、毫秒。如果提供了至少两个参数，其余的参数均会默认设置为1（如果没有提供day参数）或者0。以一个函数的形式来调用 Date 对象（即不使用 new 操作符）会返回一个代表当前日期和时间的字符串。如：

var a = Date()
console.log(a)
// Thu Jan 24 2019 11:43:48 GMT+0800 (中国标准时间)

除此之外，Date 对象还具有一下几个静态方法，分别是 Date.now()、Date.parse() 与 Date.UTC()，它们的作用是：

  • Date.now()：返回自 1970-1-1 00:00:00 UTC (世界标准时间)至今所经过的毫秒数。
  • Date.parse()：解析一个表示日期的字符串，并返回从 1970-1-1 00:00:00 所经过的毫秒数。
  • Date.UTC()：接受和构造函数最长形式的参数相同的参数（从2到7），并返回从 1970-01-01 00:00:00 UTC 开始所经过的毫秒数。

# 二、Date 的原型属性

Date 构造函数创建的对象并没有实例属性，但是在 Date 构造函数的原型对象中含有许多操作 Date 实例的方法。其中包括一系列 getter、setter 方法，可以用来操作 Date 实例的相关「属性」（事实上 Date 实例并没有实例属性），比如 getDate() 可以获取实例中的日期值，setDate()可以设置实例中的日期值。除此之外，以下「属性」都具有相应的 getter、setter 方法：

  • Date：日期对象的月份中的第几天（1-31）
  • Day：日期对象的星期中的第几天（0-6）
  • FullYear：日期对象的年份（四位数年份时返回四位数字）
  • Hours：日期对象的小时（0-23）
  • Milliseconds：日期对象的毫秒（0-999）
  • Minutes：日期对象的分钟（0-59）
  • Month：日期对象的月份（0-11）
  • Seconds：日期对象的秒数（0-59）
  • Time：从1970-1-1 00:00:00 UTC（协调世界时）到该日期经过的毫秒数，对于1970-1-1 00:00:00 UTC之前的时间返回负值
  • UTCDate：基于世界时的一个月的第几天（1-31）
  • UTCDay：基于世界时的一个星期的第几天（0-6）
  • UTCFullYear：基于世界时的所在的年份（4位数）
  • UTCHours：基于世界时的当前的小时（0-23）
  • Milliseconds：基于世界时的的毫秒数（0-999）
  • UTCMinutes：基于世界时的分钟数（0-59）
  • UTCMonth：基于世界时的月份（0-11）
  • UTCSeconds：基于世界时的秒数（0-59）

# 三、Date 对象转为字符串

除了上面的那些 getter、setter 方法之外，Date 对象的原型上还有其他处理方法，这些方法可以将 Date 对象输出为各种需要的日期字符串，毕竟直接使用 Date() 生成的字符串，比如：Thu Jan 24 2019 11:43:48 GMT+0800 (中国标准时间)，显得并不友好。

下面的方法就能够将 Date 对象的内容以默认格式输出为字符串，并且可以将对象的时间与日期内容分别输出：

var c = new Date()
console.log(c.toString())
// Thu Jan 24 2019 12:39:33 GMT+0800 (中国标准时间)
console.log(c.toDateString()) 
// Thu Jan 24 2019
console.log(c.toTimeString())
// 12:39:33 GMT+0800 (中国标准时间)
console.log(c.toUTCString()) //返回当前的格林威治时间
// Thu, 24 Jan 2019 04:39:33 GMT
