【归纳】React 基础知识

# 1、引入 React

想在页面中使用 React 很简单，只用在页面中引入 React 与 React DOM 这两个库就行了：

  <script src="https://unpkg.com/react@16/umd/react.development.js" crossorigin></script>
  <script src="https://unpkg.com/react-dom@16/umd/react-dom.development.js" crossorigin></script>
1
2

其中 React 是基础库，负责组建相关内容的支持；React DOM 是平台库，负责把组件渲染到页面上的操作。下面就是一个简单地示例，一个 <h1> 标签被将被渲染到 #root 元素中：

ReactDOM.render(
  <h1 className="greeting">Hello, world!</h1>,
  document.getElementById('root')
);

这就是 React 的最基础的使用，渲染元素到页面上。接下来就来说说 React 是如何做到在 JS 中写 html 代码的。

# 2、JSX

上面的例子还可以这样写，编译之后，JSX 表达式就变成了常规的 JavaScript 对象：

const element = <h1 className="greeting">Hello, world!</h1>;
ReactDOM.render(element, document.getElementById('root'));
1
2

这种特殊的标签语法就是 JSX（JavaScript in XML），既不是字符串也不是 HTML。这是 JavaScript 的一种扩展语法，在 React 中使用这种语法来描述 UI 信息。这种语法非常方便，当然也不是非用不可，实际上这种语法是下面这个方法的简写：

const element = React.createElement(
  'h1',
  {className: 'greeting'},
  'Hello, world!'
);

其中这样就达到了和之前例子一样的效果。因为 JSX 本质上还是 JS，所以其中的属性名是遵循驼峰命名法的。例如，在 JSX 中，class 变成了 className，tabindex 变成了 tabIndex。在 JSX 中，可以使用双引号来指定字符串字面量作为属性值也可以用花括号嵌入一个 JavaScript 表达式作为属性值:

const element1 = <div tabIndex="0"></div>;
const element2 = <img src={user.avatarUrl}></img>;

可以在 JSX 中使用花括号嵌入表达式。在下面的例子中，声明了一个名为 name 的变量，然后在 JSX 中使用 花括号 将其括起来：

const name = 'Josh Perez';
const element = <h1>Hello, {name}</h1>;

ReactDOM.render(
  element,
  document.getElementById('root')
);

JSX 标签可能包含子元素，如果是空标签，就应该像 XML 一样，使用 /> 立即闭合它。如果将 JSX 分割成多行。最好使用括号将 JSX 包裹起来，这样做可以避免分号自动插入的 bug：

const element1 = <img src={user.avatarUrl} />;
const element2 = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>

默认情况下， 在渲染之前, React DOM 会格式化(escapes) JSX中的所有值. 从而保证用户无法注入任何应用之外的代码. 在被渲染之前，所有的数据都被转义成为了字符串处理，因此起到了避免 XSS(跨站脚本) 攻击的效果。

# 3、组件和属性

组件(Components) 让你可以将整个用户界面分成独立的，可复用的小部件，并可以对每个部件进行单独的设计。从定义上来说， 组件就像JavaScript的函数。组件可以接收任意输入(称为”props”)， 并返回 React 元素，用以描述屏幕显示内容。React 元素有函数式组件和类组件两种：

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

上面的函数是一个有效的 React 组件，因为它接收一个 props 参数, 并返回一个 React 元素。 把此类组件称为函数式(Functional)组件， 因为从字面上看来它就是一个 JavaScript 函数。也可以用一个 ES6 的 class 来定义一个组件。可以这样在 JSX 中使用一个组件，就像使用 DOM 标签一样：

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

const element = <Welcome name="Sara" />;
ReactDOM.render(
  element,
  document.getElementById('root')
);

当 React 遇到一个代表用户定义组件的元素时，它将 JSX 属性以一个单独对象的形式传递给相应的组件。 将其称为 “props” 对象。比如上面的 <Welcome> 标签中的 name 属性，组件中就可以在 props 对象中获取到，并将其渲染到组件中。特别的是，可以 props.children 获取到使用组件时给这个组件设置的所有子节点。

组件名称必须以大写字母开始。当一个元素类型以小写字母开头，它表示引用一个类似于 <div> 或者 <span> 的内置组件，会给 React.createElement 方法传递 'div' 或者 'span' 字符串。以大写字母开头的类型，类似于 <Foo />，会被编译成 React.createElement(Foo) ，对应于自定义组件 或者在 JavaScript 文件中导入的组件。

Props 是只读的。所有 React 组件都必须是纯函数，并禁止修改其自身 props 。因为它们不会试图改变它们的输入，并且对于同样的输入，始终可以得到相同的结果。虽然不能直接操作 props 的值，但是可以给 props 设置默认值，还能限制 props 传入的类型。这时需要设置组件的一些属性，下面是一个例子：

class Greeting extends React.Component {
  render() {
    return (
      <h1>Hello, {this.props.name}</h1>
    );
  }
}

Greeting.defaultProps = { //设置默认值
  name: 'Mary'
};
Greeting.propTypes = {  //设置属性类型
  name: PropTypes.string
};

# 4、状态(State) 和 生命周期

为了使能够操作组件的内部状态，需要需要添加 state 到组件中。state 和 props 类似，但是它是私有的，并且由组件本身完全控制。而且只有类组件可以设置 state，因此如果要用到 state，就必须使用类组件。下面是在一个类组件添加 state 的示例：

class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}

在类组件中，还可以借助声明周期函数使得对代码更具有控制力。下面是几种 React 生命周期函数：

1. constructor()：构造函数：组件被加载前最先调用，并且仅调用一次
2. componentWillMount()：组件初始渲染前调用，并且仅调用一次
3. render()：componentWillMount之后，componentDidMount之前，渲染挂载组件
4. componentDidMount()：render之后被调用，并且仅调用一次
5. componentWillReceiveProps(nextProps)：组件渲染后，当组件接收到新的props时被调用；这个函数接收新的 props 为参数
6. shouldComponentUpdate(nextProps, nextState)：组件挂载后（即执行完render），接收到新的state或props时被调用，即每次执行setstate都会执行该函数，来判断是否重新render组件，默认返回true；接收两个参数：第一个是新的props，第二个是新的state。如果有些变化不需要重新render组件，可以在该函数中阻拦
7. componentWillUpdate()：在接收到新的props 或者 state，重新渲染 之前立刻调用，在初始化渲染的时候该方法不会被调用
8. componentDidUpdate()：重新渲染后调用，在初始化渲染的时候该方法不会被调用
9. componentWillUnmount()：组件卸载前调用，在该方法中执行任何必要的清理，比如无效的定时器，或者清除在 componentDidMount 中创建的 DOM 元素。

# 4、处理事件

通过 React 元素处理事件跟在 DOM 元素上处理事件非常相似。但是有一些语法上的区别。在React 事件使用驼峰命名，而不是全部小写；通过 JSX , 你传递一个函数作为事件处理程序，而不是一个字符串。以点击事件为例，当使用 React 时，你一般不需要调用 addEventListener 在 DOM 元素被创建后添加事件监听器。相反，只要当元素被初始渲染的时候提供一个监听器就可以了。

例如， Toggle 组件渲染一个按钮，让用户在 “ON” 和 “OFF” 状态之间切换：

class Toggle extends React.Component {
  constructor(props) {
    super(props);
    this.state = {isToggleOn: true};

    // 这个绑定是必要的，使`this`在回调中起作用
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState(state => ({
      isToggleOn: !state.isToggleOn
    }));
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        {this.state.isToggleOn ? 'ON' : 'OFF'}
      </button>
    );
  }
}

ReactDOM.render(
  <Toggle />,
  document.getElementById('root')
);

在JSX回调中你必须注意 this 的指向。 在 JavaScript 中，类方法默认没有 绑定 的。如果你忘记绑定 this.handleClick 并将其传递给onClick，那么在直接调用该函数时，this 会是 undefined 。这不是 React 特有的行为；这是 JavaScript 中的函数如何工作的一部分。 一般情况下，如果你引用一个后面没跟 () 的方法，例如 onClick={this.handleClick} ，那你就应该 绑定(bind) 该方法。

使用箭头函数可以免于绑定 this，这是利用了剪头函数 this 永远指向函数定义时所在对象。比如：

class LoggingButton extends React.Component {
  handleClick = () => {
    console.log('this is:', this);
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        Click me
      </button>
    );
  }
}

在回调中也可以使用箭头函数，比如：

class LoggingButton extends React.Component {
  handleClick() {
    console.log('this is:', this);
  }

  render() {
    return (
      <button onClick={(e) => this.handleClick(e)}>
        Click me
      </button>

}

这个语法的问题是，每次 LoggingButton 渲染时都创建一个不同的回调，这是由箭头函数是匿名函数的特性导致的。如果这个回调被作为 prop(属性) 传递给下级组件，这些组件可能触发不必要的重复渲染。因此在回调中使用箭头函数是不推荐的。通常建议在构造函数中进行绑定，以避免这类性能问题。
