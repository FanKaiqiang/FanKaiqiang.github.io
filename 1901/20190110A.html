# 用原型模拟类

在 ES6 之前，JavaScript 中并没有类的语法。可以把类看作是拥有共同特征的对象，要生成某个类的对象，就要借助相应的构造函数。通过对原型的操作，可以模拟出 ES6 中 class 语法的效果，因此 ES6 中引入的 JavaScript 类实质上是 JavaScript 现有的基于原型的继承的语法糖。

至于原型这个知识点，之前以及写过两篇博客了，一篇是在刚学原型时候记录的知识点，另一篇中其实就讲到了 ES5/6 中实现类的方法，里面其实也提到了今天的大部分知识点。这两篇博客链接如下：

  1. 【记录】原型与原型链 - 樊凯强的博客 | Franko's Blog
  2. 【记录】JS模拟继承 - 樊凯强的博客 | Franko's Blog

之前博客里记载的原型模拟类的方法大致如下。其中在构造函数中使用 this 指向类的自有属性，并且要用一个有点绕的方法实现类的继承：

 function Human(name){  //这个函数自身的属性
     this.name = name
 }
 Human.prototype.run = function(){  //原型上的属性
     console.log("我叫"+this.name+"，我在跑")
     return undefined
 }
 function Man(name){
     Human.call(this, name)//调用父类，继承父类本身的属性
     this.gender = '男'
 }

// 这样的写法是错误的
// Man.prototype.__proto__ = Human.prototype
// 下面的三行代码连接了父类与之类的原型
 var f = function(){}
 f.prototype = Human.prototype
 Man.prototype = new f()

 Man.prototype.fight = function(){
     console.log('糊你熊脸')
 }

 var person = new Man('')

# ES6 的 class 语法

类实际上是个“特殊的函数”，就像你能够定义的函数表达式和函数声明一样，类语法有两个组成部分：类表达式和类声明。直接来看一个类声明的例子，然后再分别来讲这些相应的知识点：

class Human{
    constructor(name){ //自身属性写在constructor里
        this.name = name
    }
    run(){ //原型中的共有属性与constructor并列写出
        console.log("我叫"+this.name+"，我在跑")
        return undefined
    }
}
class Man extends Human{   //使用extends关键字实现共有属性继承
    constructor(name){
        super(name)    //使用super关键字调用父类构造函数
        this.gender = '男'
    }
    fight(){
        console.log('糊你熊脸')
    }
}

使用 class 语法定义的类可以是匿名的，赋予一个命名类表达式的名称是类的主体的本地名称。需要注意的是，使用 class 声明类并没有变量提升的问题。比如：

let p = new Rectangle(); 
// ReferenceError

/* 匿名类 */ 
let Rectangle = class {
  constructor(height, width) {
    this.height = height;
    this.width = width;
  }
};

/* 命名的类 */ 
let Rectangle = class Rectangle {
  constructor(height, width) {
    this.height = height;
    this.width = width;
  }
};

在类中，使用 constructor 方法来构建类的自有属性，constructor 是一个特殊的方法，其用于创建和初始化使用 class 创建的一个对象。一个类只能拥有一个名为 “constructor” 的特殊方法。

而类的共有属性可以直接写在类的作用域中，这种方法和 ES5 操作原型的方法相比方便许多。需要注意的是这里设置的共有属性只能是函数类型的，如果要设置非函数类型的共有属性。可以通过设置共有方法但 return 一个非函数值来实现非函数属性的效果。

除此之外，在 class 中还可以设置 get、set 方法。借助这个功能同样可以实现给类添加非函数共有属性的效果。使用这种方法的好处是，在调用这个非函数属性的时候不需要像调用函数一样调用它。比如：

class Rectangle {
    // constructor
    constructor(height, width) {
        this.height = height;
        this.width = width;
    }
    // Getter
    get area() {
        return this.calcArea()
    }
    // Method
    calcArea() {
        return this.height * this.width;
    }
}
const square = new Rectangle(10, 10);

console.log(square.area);
// 100

在这个类中，get 方法控制了 area 属性的读取，如果没有为这个属性设置了 set 方法，那么这个属性就是只读属性。set 方法接受属性的赋值为参数，借此可以达到筛选赋值的效果。get、set 的内容在之前的博客也在这里提到过：【归纳】JS 对象的一些知识 - 樊凯强的博客 | Franko's Blog，这里就不赘述了。

在类中，使用 extends 关键字在类声明或类表达式中用于创建一个类作为另一个类的一个子类，如果子类中存在构造函数，则需要在使用 “this” 之前首先调用 super()，super 关键字还可以用于调用对象的父对象上的函数。可以把这种继承方式看作是 ES5 中「三句代码实现继承」方法的语法糖，事实上，extends 同样可以用在传统的基于函数的“类”的继承上：

function Animal (name) {
  this.name = name;  
}
Animal.prototype.speak = function () {
  console.log(this.name + ' makes a noise.');
}

class Dog extends Animal {
  speak() {
    super.speak();  //使用super 关键字用于调用对象的父对象上的函数。
    console.log(this.name + ' barks.');
  }
}

var d = new Dog('Mitzie');
d.speak();

在类中，还可以使用 static 关键字用来定义一个类的一个静态方法。调用静态方法不需要实例化该类，但不能通过一个类实例调用静态方法。静态方法通常用于为一个应用程序创建工具函数。实际上，为类添加静态方法即相当于直接在构造函数上添加属性，下面是一个例子，直接在 Point 类上就能访问到静态方法：

class Point {
    constructor(x, y) {
        this.x = x;
        this.y = y;
    }

    static distance(a, b) {
        const dx = a.x - b.x;
        const dy = a.y - b.y;

        return Math.hypot(dx, dy);
    }
}

const p1 = new Point(5, 5);
const p2 = new Point(10, 10);

console.log(Point.distance(p1, p2));

类的 Symbol.species 属性允许你覆盖默认的构造函数。例如，当使用像map()返回默认构造函数的方法时，您希望这些方法返回一个父Array对象，而不是MyArray对象。Symbol.species 符号可以让你这样做：

class MyArray extends Array {
  // Overwrite species to the parent Array constructor
  static get [Symbol.species]() { return Array; }
}
var a = new MyArray(1,2,3);
var mapped = a.map(x => x * x);

console.log(mapped instanceof MyArray); 
// false
console.log(mapped instanceof Array);   
// true

一个 ECMAScript 类只能有一个单超类，如果要实现多重继承，可以创建一个以超类作为输入的函数并将一个继承该超类的子类作为输出：

var calculatorMixin = Base => class extends Base {
  calc() { }
};

var randomizerMixin = Base => class extends Base {
  randomize() { }
};

class Foo { }
class Bar extends calculatorMixin(randomizerMixin(Foo)) { }
