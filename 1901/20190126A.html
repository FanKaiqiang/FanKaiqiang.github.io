【归纳】async 和 await

广宇 2019-1-27 JavaScript

这个月我只能尽力写到 10 篇吧，因为这个月我还想把 vue 毕设给看掉，这件事同样很紧迫。这样到了二月，我才有更多时间好好打磨自己。

本篇博客的主题是 async 和 await。这部分的知识之前我也算是没学多久吧，属于不太清楚的内容，在毕设阶段才正式接触，而这回正好 ES6 深入浅出也加了一节相关课程，也算是雪中送炭了。刚刚把这节课给看了，正好结合之前在毕设中学到的相关知识也一起消化了，整理整理笔记，顺便复习一下 Promise 的相关内容，边写博客边把 MDN ：async function | MDN 过一遍。

# 一、async function 的实例

要想了解一个自己并不清楚的概念，我会倾向于查找 MDN 中的相关内容。因为 async 是与 function 联合使用的，所以在 MDN 中，我只找到了 await function 的定义：

async function 声明用于定义一个返回 AsyncFunction 对象的异步函数。异步函数是指通过事件循环异步执行的函数，它会通过一个隐式的 Promise 返回其结果。但是如果你的代码使用了异步函数，它的语法和结构会更像是标准的同步函数。

可能是上面的提到的一些概念我并不清楚，也有可能是翻译的原因，这段话看的我并不清楚。但是基本可以从其中确定两点：async 是与 Promise 结合使用的；async 可以使异步函数看起来像同步函数。直接来看一个简单的示例：

function resolveAfter2Seconds() {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve('resolved');
    }, 2000);
  });
}

async function asyncCall() {
  console.log('calling');
  var result = await resolveAfter2Seconds();
  console.log(result);
}

asyncCall();

resolveAfter2Seconds().then((result) => {

这一段就看的懂了，resolveAfter2Seconds 返回的 Promise 两秒后状态变为 resolve，并将 'resolved' 作为操作结果传递给 then，then 拿到结果之后输出。实际上，两种操作的效果都是一样的，那么这里的 async function 到底是什么，又起到了什么样的作用呢？这里就要先讲到这个函数里面的 await 了。

# 二、await是什么

这节就来讲讲讲讲 await。就上面的例子而言，await 后面接了一个函数的返回值，这个返回值是 Promise 对象，然后这个 await 操作被赋值给 result。如果光让自己看代码，绝对会感到一头雾水，实际上 await 的作用正与它的名称一样，是等待的意思。

来看看 MDN 中的意思：

await 表达式会暂停当前 async function 的执行，等待 Promise 处理完成。若 Promise 正常处理(fulfilled)，其回调的resolve函数参数作为 await 表达式的值，继续执行 async function。

这句话讲的就很清楚， await 后面接一个 Promise 对象就相当于 Promise.then()，因为这个Promise的成功回调会作为 await 的结果，是可以赋值给其他变量的。await 会暂停当前 async function 的执行，使得函数可以等待异步操作完成之后再继续执行，这就达到了想执行同步操作那样执行异步操作。

需要指出的是，只有 Promise 回调的 resolve 函数参数才会作为 await 的值。若 Promise 处理异常(rejected)，await 表达式会把 Promise 的异常原因抛出。使用 try/catch 可以捕获这个错误，起到相当于 Promise.catch() 的效果：

async function f1() {
  try {
    var z = await Promise.reject(30);
  } catch (e) {
    console.log(e); // 30
  }
}
f1();

另外，如果 await 操作符后的表达式的值不是一个 Promise，await 会把该值转换为已正常处理的Promise，然后则返回该值本身。

async function f2() {
  var y = await 20;
  console.log(y); // 20
}
f2();

# 三、回过头来讲 async

上面也列了好几个不同的 async function 的例子，可以发现，async 更像是一种和函数声明相关的关键字。async 是 asynchronous 的简写，即异步。声明函数时如果加上 async 的前缀，就是在声明异步函数。

当调用一个 async 函数时，会返回一个 Promise 对象。当这个 async 函数返回一个值时，Promise 的 resolve 方法会负责传递这个值；当 async 函数抛出异常时，Promise 的 reject 方法也会传递这个异常值。以上一个例子为例，尽管这个 f2() 中并没有什么异步操作，但是 f2() 的返回值是一个 Promise 对象，达到了声明异步函数的目的：

async function f2() {
  var y = await 20;
}
console.log(f2())
//  Promise {<pending>}
f2().then((x)=>{console.log(x)})
//  20

换句话说，就是 async function 的返回值将被隐式地传递给了 Promise.resolve()

上面还讲到过，async 函数中可能会有 await 表达式，这会使 async 函数暂停执行，等待 Promise 的结果出来，然后恢复async函数的执行并返回解析值（resolved）。await 也必须在 async function 中使用。

可以发现，所有 async/await 的功能都是围绕着 Promise 进行的，async/await 能做的，Promise 也都能做，那么使用 async/await 的意义何在呢？从上面的例子可以看出，所有 async function 的执行顺序就是从上往下，像是标准的同步函数，根本不像是 JavaScript 传统的异步操作。在我刚学 JS 的时候，异步让我摸不着头脑，而借助这种语法，就能写出「同步」的异步代码，在逻辑上更加清晰了。借用阮一峰 async 相关博客里的一句话来说，那就是：

异步编程的最高境界，就是根本不用关心它是不是异步。

当然，async 也不是全能。比如，await 表达式后面只能接受一个 Promise，这意味着 await 不能同时对两个 Promise 对象进行处理。下面来看两个例子，不过这两个例子都不是同时处理 Promise 的，但是这也能够帮助更好地理解 async：

var sequentialStart = async function() {
  console.log('==SEQUENTIAL START==');

  // 如果 await 操作符后的表达式不是一个 Promise 对象, 则它会被转换成一个 resolved 状态的 Promise 对象
  const slow = await resolveAfter2Seconds();

  const fast = await resolveAfter1Second();
  console.log(slow);
  console.log(fast);
}

var concurrentStart = async function() {
  console.log('==CONCURRENT START with await==');
  const slow = resolveAfter2Seconds(); // 立即启动计时器
  const fast = resolveAfter1Second();

  console.log(await slow);
  console.log(await fast); // 等待 slow 完成, fast 也已经完成。
}

虽然 async 与 Promise.then() 相似，但是不要将 await 和 Promise.then() 混淆。在sequentialStart中，程序为第一个await停留了2秒，然后又为第二个await停留了1秒。直到第一个计时器结束后，第二个计时器才被创建。

在 concurrentStart中，两个计时器均被创建，然后一起被await。这两个计时器同时运行的，但await的调用仍然是按顺序运行的，这意味着第二个 await 会等到第一个 await 运行结束。这使得代码完成运行只需要2秒，而不是3秒。这2秒是由最慢的计时器决定的。

如果一定要使用 await 同时处理两个 Promise 对象，只能借助 Promise.all()，因为 Promise.all() 方法的返回值也是一个 Promise 对象，这个方法也在之前的 Promise 博客里介绍过。正如：

let x = await Promise.all(promise1,promise2)
